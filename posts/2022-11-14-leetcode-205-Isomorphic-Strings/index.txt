11:T1186,<p>굉장히 쉬운 문제 였는데, 너무 어렵게 생각해서 시간 낭비한 문제 입니다. 🫠</p>
<h3>문제</h3>
<blockquote>
<p><a href="https://leetcode.com/problems/isomorphic-strings/description/?envType=study-plan&#x26;id=level-1">205. Isomorphic Strings 문제 링크</a></p>
</blockquote>
<p>같은 모양의 문자열인지 판별하시오!
<br>
(단! 한 문자는 한 문자만 대체 가능!)</p>
<pre><code class="hljs language-text"># Example 1:
Input: s = "egg", t = "add"
Output: true

# Example 2:
Input: s = "foo", t = "bar"
Output: false
</code></pre>
<p>예를 들어 예제 1번(egg 와 add)
<br>
s(egg)를 e → a, g → d로 대체 하면 둘은 같은 문자가 됩니다.</p>
<p>하지만 조건으로 한 문자는 한 문자만 대체 가능하므로...
<br>
예제 2번과 같이 s를 변환 하려고 해도 o → a 로 이미 정해져 있으므로 변환을 해도 <code>baa &#x3C;> bar</code>로 일치하지 않게 됩니다.
<br>
반대로 t → s로 변환하려고 해도? 🤔
<br>
<code>{ b: f, a: o, r: ?????}</code> 와 같이 a - o 는 이미 매칭되어 r - o는 매칭 될수 없기 때문에 변환 실패입니다.</p>
<hr>
<h3>처음 작성한 풀이</h3>
<ol>
<li>❎ 단순히 replace() 사용
  <br>
</li>
</ol>
<p>→ 한 단어는 하나만 변환 가능 조건이 안붙어서 무조건 True 만 반환하여 실패
2. ❎ 문자를 처음 인덱스로 변환해서 각 합이 같은지?
<br>
→ <code>s = bbbaaaba, t = aaabbbba</code> 라면 순서만 바뀌는 것이기 때문에 무조건 합이 같아 True 만 반환하여 실패
3. ✅ 변환 사전을 만들어서 변환! (아래 작성 코드)</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">def</span> <span class="hljs-title function_">isIsomorphic</span>(<span class="hljs-params">self, s: <span class="hljs-built_in">str</span>, t: <span class="hljs-built_in">str</span></span>) -> <span class="hljs-built_in">bool</span>:
  <span class="hljs-keyword">if</span> <span class="hljs-built_in">len</span>(<span class="hljs-built_in">set</span>(s)) != <span class="hljs-built_in">len</span>(<span class="hljs-built_in">set</span>(t)):
    <span class="hljs-keyword">return</span> <span class="hljs-literal">False</span>

  dic = {}
  encoded_s = <span class="hljs-string">''</span>
  <span class="hljs-keyword">for</span> i <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(<span class="hljs-built_in">len</span>(s)):
    <span class="hljs-keyword">if</span> s[i] <span class="hljs-keyword">not</span> <span class="hljs-keyword">in</span> dic.keys():
      dic[s[i]] = t[i]
    encoded_s += dic[s[i]]
  <span class="hljs-keyword">return</span> t == encoded_s
</code></pre>
<p>한 단어당 하나만 대체 가능하다는 조건을 위해 set 자료형을 사용해서 문자 종류 수 가 같은지 체크하게 됩니다.
<br>
그리고 for 문으로 하나씩 변환하여 두 단어가 같은지 체크하는 방식으로 검사했습니다.</p>
<h3>최종 Solution</h3>
<p>하지만 더 간단한 방법이 있었는데요. 😓
<br>
위의 2번에서 굳이 합산을 해서 비교하는 게 아닌 단순하게 리스트로 비교를 하게 되면 끝나는 거였습니다!
<br>
(1문자당 1문자 대체 가능! &#x26; 단어의 순서까지 체크가 가능하게 되는 거죠.)
<br>
(ex) bbbaaaba : aaabbbba = 00011101 : 00011110 → ✅ False 반환!</p>
<p>파이썬은 리스트 자체로 비교가 가능하다는 장점을 간과했습니다. 🫠</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">def</span> <span class="hljs-title function_">isIsomorphic</span>(<span class="hljs-params">self, s: <span class="hljs-built_in">str</span>, t: <span class="hljs-built_in">str</span></span>) -> <span class="hljs-built_in">bool</span>:
  index_s = []
  index_t = []
  <span class="hljs-keyword">for</span> word <span class="hljs-keyword">in</span> s:
    index_s.append(s.index(word))
  <span class="hljs-keyword">for</span> word <span class="hljs-keyword">in</span> t:
    index_t.append(t.index(word))
  <span class="hljs-keyword">return</span> index_s == index_t
</code></pre>
<p>처음 작성한 코드와 사실 실행 시간은 같습니다.
<br>
