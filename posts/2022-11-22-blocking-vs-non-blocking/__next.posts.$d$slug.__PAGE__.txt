<h2>Blocking vs Non-Blocking</h2>
<p>둘의 차이점의 키-포인트는 <em><strong>제어권</strong></em> 입니다.</p>
<pre><code class="hljs language-text">함수 A() {
  B();
  // 다음 작업!
}

함수 B() {
  // 열심히 실행!
}
</code></pre>
<blockquote>
<p>A 함수 내에서 B 함수 실행 시?</p>
</blockquote>
<h3>⛔️ 블로킹 (Blocking)</h3>
<p>A는 B에게 제어권을 넘겨주고, 함수 실행을 잠시 멈춥니다(Block!).
<br>
B는 함수를 실행하고 A에게 제어권을 넘겨줍니다.
<br>
A는 제어권을 넘겨받고 나서 다음 작업을 이어갑니다.</p>
<p><code>Javascript</code>에서 <code>alert()</code> 함수로 설명해보겠습니다.</p>
<pre><code class="hljs language-javascript"><span class="hljs-title function_">alert</span>(<span class="hljs-string">'Hi!'</span>);
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'Bye!'</span>);
</code></pre>
<p>위의 작업의 결과는 보이는 바와 같이 아래와 같습니다.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_"># </span><span class="bash">메시지 알림으로 Hi! 팝업</span>
Bye!
</code></pre>
<p>메시지 팝업으로 <code>Hi!</code>를 보여준 후에 <code>Bye!</code>가 콘솔로 출력됩니다.
<br>
이는 <code>alert</code> 함수로 제어권이 넘어가서 실행을 완료되기 전까지 Block 됩니다.
<br>
메시지 창을 닫고 나서야 제어권이 다시 돌아와 다음 작업을 실행하게 되는 것이죠.</p>
<h3>⛩️ 논블로킹 (Non-blocking)</h3>
<p>A에서 B를 호출하지만 B가 바로 제어권을 반납하여 A가 제어권을 가지게 됩니다.
<br>
A는 B를 호출하고도 계속해서 다음 작업을 실행합니다.</p>
<p>똑같이 <code>Javascript</code>로 예시를 들어보겠습니다.
<br>
비동기 함수로 사용되는 <code>setTimeout()</code>로 설명해볼게요.</p>
<pre><code class="hljs language-javascript"><span class="hljs-built_in">setTimeout</span>(<span class="hljs-function">() =></span> 
  <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'Hi!'</span>), <span class="hljs-number">0</span>); <span class="hljs-comment">// 2번째 매개변수: 대기 시간을 0으로 설정!</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'Bye!'</span>);
</code></pre>
<p>위의 작업을 실행하게 되면 어떻게 될까요?
<br>
정답은 아래와 같습니다.</p>
<pre><code class="hljs language-shell">Bye!
Hi!
</code></pre>
