생각을 해보니.. 성능 개선을 위한 렌더링 순서 관련된 내용을 들은 기억이 나서 그 내용도 추가!
<h2>✨ 성능을 위한 렌더링 순서 이해하기 (with Critical rendering path)</h2>
<blockquote>
<p>requests/response → loading → scripting → rendering → layout → painting</p>
</blockquote>
<ol>
<li>requests/response
  <br>
</li>
</ol>
<p>브라우저가 서버에게 html 파일을 요청,
<br>
서버에 request 하면 가장 먼저 index.html 받음
<br>
→ 필요한 링크 파일(CSS, JS) 받음
<br>
→ 필요한 리소스(image, font) 순차적으로 서버에서 받음
3. loading
<br>
html 파일을 서버에서 받아 로딩
3. scripting
<br>
html 파일을 한줄 한줄 읽어 DOM 요소로 변환 (DOM Tree 생성),
<br>
CSS 스타일을 모두 계산 → CSSOM Tree 생성
4. rendering
<br>
DOM + CSSOM → 브라우저에 표기될 요소 선별 → <strong>Render Tree</strong> 생성
5. layout
<br>
Render Tree 각 요소들이 어떤 위치에 얼마나 크게 차지 할것인지 계산!
6. painting
<br>
그리기</p>
<h3><strong>어떤 일을 하는지 카테고리로 나누어 생각을 해보면? 🤔</strong></h3>
<ul>
<li>Construction</li>
<li>만들어진 렌더링 트리를 이용해 구조를 작성하고 어디에 배치할건지 계산을 한 다음 실제로 브라우저 윈도우에 그림을 그려주는, 렌더링을 하주는 Operation</li>
</ul>
<ol>
<li>🏗️ Construction : HTML 페이지에서 브라우저가 이해할 수 있도록 브라우저 언어로 바꾸는 작업</li>
<li>🎨 Operation : <code>Rendering Tree</code>를 이용해서 구조와 배치 계산 후, 실제로 그림을 그리는(렌더링) 작업</li>
</ol>
<blockquote>
<p>DOM > CSSDOM > Render Tree > layout > paint > composition</p>
</blockquote>
<h3>🏗️ Construction</h3>
<blockquote>
<p>DOM > CSSDOM > Render Tree</p>
</blockquote>
<p><strong>💁‍♂️ DOM 요소가 작으면 작을 수록, CSS 규칙이 작으면 작을 수록 빨라집니다.</strong></p>
<p>고로 불필요한 태그를 쓴다던지(div 태그 남용),
<br>
쓸데없이 래핑 클래스나 래핑 요소를 만든다던지는 자제해야 합니다.</p>
<h3>✨ Operation</h3>
<blockquote>
<p>layout > paint > composition</p>
</blockquote>
<p>생성된 Render Tree를 이용하여 브라우저 window에 표기할 요소의 위치, 크기 등을 계산하고 그림을 그리는 단계</p>
<ol>
<li>layout
  <br>
</li>
</ol>
<p>각 요소를 어떤 크기로 어디에 위치시킬 것인지 <strong>구성과 배치</strong>
2. paint
<br>
그림을 바로 그리는 것이 아닌! 🙅‍♂️
<br>
요소들을 어떻게 배치했는지에 따라 각 부분을 잘게 잘게 나누어 이미지(BITMAP)를 준비해 놓는 단계
<br>
(다양한 속성값에 따라 브라우저 엔진마다 성능 개선을 위해 레이어를 준비)
<br>
<br>
이는 변경 사항이 있을 때, 전체를 다시 그리는 것이 아닌 해당 부분만 수정할 수 있도록 하기 위함!
<br>
(그렇다고 해서 레이어를 불필요하게 너무 잘게 나누어 많아져도 성능 문제 발생되니 남용 금지)
3. composition
<br>
페인트에서 준비한 레이아웃들을 합치는 단계</p>
<blockquote>
<p>🪄 여기서 중요한 것은!
<br>
layout > paint > composition 순서로 진행되기 때문에,</p>
<ul>
<li><strong>composition 만 일어난다면 Best!</strong> 👍
<br>
</li>
<li>paint 가 일어난다면 그럭 저럭? 🤔
<br>
</li>
<li>layout 이라면... 성능상 문제가 생길 여지가 다분함! 😓</li>
</ul>
</blockquote>
<p>그래서 위치를 이동 시킬 때, <code>top, left</code>가 아닌 <code>transform</code>을 사용하는 것이 좋습니다.
