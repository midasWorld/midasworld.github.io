<h2>PriorityQueue vs heapq</h2>
<p>PriorityQueue vs heapq 차이점을 알아보겠습니다.</p>
<h3>CPython - PriorityQueue 클래스 일부</h3>
<pre><code class="hljs language-python"><span class="hljs-keyword">class</span> <span class="hljs-title class_">PriorityQueue</span>:
    ...
    <span class="hljs-keyword">def</span> <span class="hljs-title function_">_put</span>(<span class="hljs-params">self, item</span>):
        heappush(<span class="hljs-variable language_">self</span>.queue, item)
        
    <span class="hljs-keyword">def</span> <span class="hljs-title function_">_get</span>(<span class="hljs-params">self</span>):
        <span class="hljs-keyword">return</span> heappop(<span class="hljs-variable language_">self</span>.queue)
</code></pre>
<h3>차이점은 스레드 세이프(Thread-Safe)</h3>
<blockquote>
<p>스레드 세이프(Thread-Safe)?
<br>
멀티 스레드에도 안전한 프로그래밍의 개념.
<br>
만약 스레드 세이프하지 않은 경우 1번 스레드의 값이 2번 스레드에서 변경될 수 있어 문제가 발생한다.</p>
</blockquote>
<p>파이썬은 GIL의 특성상 멀티 스레딩이 거의 없기 때문에 대부분 멀티 프로세싱으로 활용한다는 점을 생각해본다면...
<br>
멀티 스레딩 지원은 사실 큰 의미를 없습니다.
<br>
또한 스레드 세이프를 보장한다는 것은 내부적으로 락킹(Locking)을 제공한다는 의미로
<br>
락킹 오버헤드(Locking Overhead)가 발생해 성능에 영향을 끼칩니다.</p>
<p>따라서 굳이 멀티 스레드로 구현할 것이 아니라면 <code>PriorityQueue</code> 모듈을 사용할 필요가 없습니다.
<br>
(실무에서도 우선순위 큐는 대부분 <code>heapq</code>로 구현을 하고 있다고 합니다. 🤔)</p>

<h3>아스테리스크(*)</h3>
<p>C 언어의 포인터 변수와 혼동될 수 있으나 <strong>파이썬에는 포인터가 존재하지 않습니다.</strong>
<br>
파이썬에서 *는 언팩(Unpack) 입니다.
<br>
시퀀스 언패킹 연산자(Sequence Unpacking Operator)로 말 그대로 시퀀스를 풀어헤치는 연산자입니다.
<br>
주로 튜플이나 리스트를 언패킹하는 데 사용됩니다.</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">import</span> collections

nums = [<span class="hljs-number">1</span>, <span class="hljs-number">1</span>, <span class="hljs-number">1</span>, <span class="hljs-number">2</span>, <span class="hljs-number">2</span>, <span class="hljs-number">3</span>]
result = collections.Counter(nums).most_common(<span class="hljs-number">2</span>)

<span class="hljs-built_in">print</span>(result) <span class="hljs-comment"># 1</span>
[(<span class="hljs-number">1</span>, <span class="hljs-number">3</span>), (<span class="hljs-number">2</span>, <span class="hljs-number">2</span>)]

<span class="hljs-built_in">print</span>(<span class="hljs-built_in">list</span>(<span class="hljs-built_in">zip</span>(result))) <span class="hljs-comment"># 2</span>
[((<span class="hljs-number">1</span>, <span class="hljs-number">3</span>),), ((<span class="hljs-number">2</span>, <span class="hljs-number">2</span>),)]

<span class="hljs-built_in">print</span>(<span class="hljs-built_in">list</span>(<span class="hljs-built_in">zip</span>(*result))) <span class="hljs-comment"># 3</span>
[(<span class="hljs-number">1</span>, <span class="hljs-number">2</span>), (<span class="hljs-number">3</span>, <span class="hljs-number">2</span>)]
</code></pre>
<p>위와 같이 Counter... 의 결과를 <code>zip()</code>으로 바로 묶으면 2번과 같이 엉뚱한 결과가 나오게 됩니다.
<br>
이는 *로 언패킹을 해줘야 원하는 값을 얻을 수 있습니다.</p>
<p>예제를 더 살펴보겠습니다.</p>
<pre><code class="hljs language-python">fruits = [<span class="hljs-string">'lemon'</span>, <span class="hljs-string">'pear'</span>, <span class="hljs-string">'watermelon'</span>, <span class="hljs-string">'tomato'</span>]

<span class="hljs-keyword">for</span> f <span class="hljs-keyword">in</span> fruits: <span class="hljs-comment"># 1</span>
    <span class="hljs-built_in">print</span>(f, end=<span class="hljs-string">' '</span>)
    
<span class="hljs-built_in">print</span>(*fruits) <span class="hljs-comment"># 2</span>
</code></pre>
<p>각 요소를 출력하는 방법은 1번과 같이 for 문을 사용해야 하지만...
<br>
놀랍게도 *로 언패킹을 해줘도 같은 결과가 출력됩니다!</p>
<p>이외에도 *는 활용도가 많다고 하는데요.</p>
<pre><code class="hljs language-python">date_info = {<span class="hljs-string">'year'</span>: <span class="hljs-string">'2020'</span>, <span class="hljs-string">'month'</span>: <span class="hljs-string">'01'</span>, <span class="hljs-string">'day'</span>: <span class="hljs-string">'7'</span>}
new_info = {**date_info, <span class="hljs-string">'day'</span>: <span class="hljs-string">'14'</span>}

<span class="hljs-built_in">print</span>(new_info)
{<span class="hljs-string">'year'</span>: <span class="hljs-string">'2020'</span>, <span class="hljs-string">'month'</span>: <span class="hljs-string">'01'</span>, <span class="hljs-string">'day'</span>: <span class="hljs-string">'14'</span>}
</code></pre>
<p>위와 같이 <code>**date_info</code>에 모든 요소를 언패킹 할 수 있고,
<br>
<code>day</code>가 14로 업데이트 된것을 확인할 수 있습니다.</p>
<h2>객체 복사</h2>
<blockquote>
<p>파이썬의 중요한 특징 중 하나!
<br>
모든것이 객체이다!</p>
</blockquote>
<p>위와 글과 같이 파이썬은 모든 것이 객체 입니다.
<br>
숫자, 문자까지도 객체죠. 🕺</p>
<p>숫자, 문자가 리스트, 딕셔너리와 같은 객체와 차이점은 <strong>불변 객체</strong>라는 것 뿐입니다.
<br>
그래서 별도로 값을 복사하지 않으면, 변수에 값을 할당하는 모든 행위는 값 객체에 대한 참조가 됩니다.
<br>
이는 즉 참조가 가리키는 원래의 값을 변경하면 모든 참조, 모든 변수의 값 또한 함께 변경됨을 말합니다.</p>
<p>그렇다면 참조가 되지 않도록 값 자체를 복사하는 방법은 무엇이 있을까요? 🤔
<br>
가장 간단한 방법이 <code>[:]</code> 슬라이스를 사용하는 것입니다.</p>
<pre><code class="hljs language-python">a = [<span class="hljs-number">1</span>, <span class="hljs-number">2</span>, <span class="hljs-number">3</span>]
b = a
c = a[:]
<span class="hljs-built_in">print</span>(<span class="hljs-built_in">id</span>(a), <span class="hljs-built_in">id</span>(b), <span class="hljs-built_in">id</span>(c))
<span class="hljs-comment"># 4312361728 4312361728 4312236800</span>
</code></pre>
<p>[:]로 처리한 변수 c는 다른 ID를 갖는 것을 알 수 있습니다.
<br>
즉 참조 처리된 b는 a와 동일한 ID를 갖지만, 변수 c는 값 자체가 복사되어 새로운 객체가 되었습니다.</p>
<p>이외에도 <code>copy()</code> 메서드를 사용하는 방법이 있습니다.
<br>
(좀 더 직관적임 💁‍♂️)</p>
<pre><code class="hljs language-python">a = [<span class="hljs-number">1</span>, <span class="hljs-number">2</span>, <span class="hljs-number">3</span>]
b = a
c = a[:]
d = a.copy()
<span class="hljs-built_in">print</span>(<span class="hljs-built_in">id</span>(a), <span class="hljs-built_in">id</span>(b), <span class="hljs-built_in">id</span>(c), <span class="hljs-built_in">id</span>(d))
<span class="hljs-comment"># 4344310272 4344310272 4344185408 4344308800</span>
</code></pre>
<p>위와 같이 변수 d 또한 다른 ID를 갖는 것을 확인할 수 있었습니다.
<br>
(== 값이 복사되어 새로운 객체가 생성되었다.)</p>
<p>간단한 리스트의 경우에는 위와 같이 <code>[:]</code>와 <code>copy()</code>로 할 수 있지만,
<br>
복잡한 리스트의 경우에는 다음과 같이 <code>copy.deepcopy()</code> 사용해야 합니다.</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">import</span> copy
a = [<span class="hljs-number">1</span>, <span class="hljs-number">2</span>, [<span class="hljs-number">3</span>, <span class="hljs-number">5</span>], <span class="hljs-number">4</span>]
b = a
c = a[:]
d = copy.deepcopy(a)
<span class="hljs-built_in">print</span>(<span class="hljs-built_in">id</span>(a), <span class="hljs-built_in">id</span>(a[<span class="hljs-number">2</span>]), a) <span class="hljs-comment"># 4376593536 4376586048 [1, 2, [3, 5], 4]</span>
<span class="hljs-built_in">print</span>(<span class="hljs-built_in">id</span>(b), <span class="hljs-built_in">id</span>(b[<span class="hljs-number">2</span>]), b) <span class="hljs-comment"># 4376593536 4376586048 [1, 2, [3, 5], 4]</span>
<span class="hljs-built_in">print</span>(<span class="hljs-built_in">id</span>(c), <span class="hljs-built_in">id</span>(c[<span class="hljs-number">2</span>]), c) <span class="hljs-comment"># 4376586880 4376586048 [1, 2, [3, 5], 4]</span>
<span class="hljs-built_in">print</span>(<span class="hljs-built_in">id</span>(d), <span class="hljs-built_in">id</span>(d[<span class="hljs-number">2</span>]), d) <span class="hljs-comment"># 4376586624 4376587584 [1, 2, [3, 5], 4]</span>
</code></pre>
<p>자.. 위의 것을 차근차근 살펴보면... 🫠
<br></p>
<ol>
<li><code>=</code> 연산자로 생성된 b는 a, [3, 5]까지 주소가 같은 것을 확인할 수 있습니다.</li>
<li><code>[:]</code> 메서드로 생성된 c는 a는 주소가 다르지만, <strong>내부의 [3, 5]는 주소가 같은 것을 확인할 수 있습니다.</strong></li>
<li><code>copy.deepcopy()</code>로 생성된 d가 되어서야, a 뿐만 아니라 [3, 5]도 주소가 새롭게 생성된 것을 확인할 수 있습니다.</li>
</ol>
<blockquote>
<p>즉 2차원 리스트 이상의 경우 객체를 복사하기 위해서는 <code>copy.deepcopy()</code>를 사용해야 합니다.</p>
