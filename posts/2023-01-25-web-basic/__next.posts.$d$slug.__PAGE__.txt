<h1><code>http://www.example.com</code>에 입력했을 때 일어나는 일은?</h1>
<ol>
<li>사용자가 브라우저에 URL(<code>http://www.example.com</code>) 입력</li>
<li>DNS 서버에 도메인 네임으로 서버의 IP 를 찾음</li>
<li>IP 주소로 웹 서버에 TCP 3 handshake로 연결 수립</li>
<li>클라이언트는 웹 서버로 HTTP 요청 메시지를 보냄</li>
<li>웹 서버는 HTTP 응답 메시지를 보냄</li>
<li>도착한 HTTP 응답 메시지는 웹 페이지 데이터로 변환되고, 웹 브라우저에 출력</li>
</ol>
<h2>도메인 구성 요소 알아보기</h2>
<h3><code>http://www.example.com:80/posts?limit=5&#x26;cursor=10</code></h3>
<ul>
<li><code>http</code> : Protocol</li>
<li><code>www.example.com</code> : 도메인 네임</li>
<li><code>:80</code> : Port</li>
<li><code>/posts</code> : Path to the file (웹 서버 자원 경로)</li>
<li><code>?limit=5&#x26;cursor=10</code> : Query, Parameter (웹 서버에서 제공하는 추가 Parameter)</li>
</ul>
<h2>URL vs URI</h2>
<blockquote>
<p>URL은 URI이지만, 모든 URI가 URL인 것은 아니다.</p>
</blockquote>
<p>URL은 Uniform Resource Locator로 인터넷 상 자원의 위치를 의미합니다.
<br>
반면 URI는 Uniform Resource Identifier로 인터넷 상의 자원을 식별하기 위한 문자열 구성으로,
<br>
URI는 URL을 포함하게 됩니다. (URI = 식별자, URL = 식별자 + 위치)</p>
