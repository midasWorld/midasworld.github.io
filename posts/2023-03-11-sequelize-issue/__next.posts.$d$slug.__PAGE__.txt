<h1>ISSUE 정리</h1>
<h2>INSERT 시, FK(외래키) 정보가 등록 안되는 문제</h2>
<p>글(Tweet) 생성 시, 작성자 정보 등록을 위해 회원의 Id를 외래키로 설정을 하고,
<br>
시퀄라이즈에서 제공하는 <code>model.create</code> 메서드를 사용해서 등록을 해줬는데?
<br>
외래키 정보만 쏙 빠진채로 등록되는 문제가 발생했습니다. 🤔</p>
<p>아래에는 순서대로 <code>[1]외래키 설정</code>, <code>[2]실행된 테이블 생성문</code>, <code>[3]등록 메서드(create)</code>, <code>[4]등록 결과물</code> 입니다.</p>

<p>4번과 같이 user_id 는 제외된 상태로 INSERT 된것을 확인할 수 있었습니다. 🫠

<pre><code class="hljs language-javascript"><span class="hljs-keyword">export</span> <span class="hljs-keyword">async</span> <span class="hljs-keyword">function</span> <span class="hljs-title function_">create</span>(<span class="hljs-params">text, userId</span>) {

<p>위와 같이 <code>user_id</code>를 명시해줘도 똑같이 등록이 되지 않았습니다.</p>
<h3>✨ 해결</h3>
<p>이리저리 찾아보다 외래키 설정하는 부분을 보고, 설정값을 넣어줬더니 너무 간단하게 해결되었습니다.</p>
<pre><code class="hljs language-javascript"><span class="hljs-title class_">Tweet</span>.<span class="hljs-title function_">belongsTo</span>(<span class="hljs-title class_">User</span>, {
  <span class="hljs-attr">foreignKey</span>: { <span class="hljs-attr">name</span>: <span class="hljs-string">"userId"</span>, <span class="hljs-attr">allowNull</span>: <span class="hljs-literal">false</span> },
});
</code></pre>
<p>위와 같이 name 설정해주면서 겸사겸사 null 허용이 되지 않도록 같이 설정했습니다.
