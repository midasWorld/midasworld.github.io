<p>DNS (Domain Name System)란...?</p>
</blockquote>
<p>웹사이트를 찾아가기 위해 IP(ex: 192.0.2.1)를 일일이 외우는것은 매우 힘듭니다.
이를 위해 도메인 이름(<a href="http://www.example.com)%EC%9D%84">www.example.com)을</a> 통해서 IP주 주소로 접속하기 위한 시스템 입니다.</p>
<h2>재귀적 질의 vs 반복적 질의</h2>
<p>도메인 주소로 IP 주소를 알아내기 위한 방식에는 2가지가 있습니다.</p>
<h3>1. 재귀적 질의</h3>
<ol>
<li><a href="http://www.example.com">www.example.com</a> 접속</li>
<li>클라이언트는 Local DNS 서버를 통해서 Root DNS 서버에 요청,</li>
<li>Root DNS Server는 IP 주소가 자신의 서버에 등록되었는지 검사합니다.</li>
<li>등록되어 있지 않다면, TLD DNS Server.. 여기도 없으면 또다시 아래층 서버로 질의합니다.</li>
</ol>
<p>이렇게 재귀적으로 해당 IP 주소를 가지고 있는 서버까지 요청을 해서 응답 받는 방식 입니다.</p>
<h3>2. 반복적 질의</h3>
<ol>
<li><a href="http://www.example.com">www.example.com</a> 접속</li>
<li>클라이언트는 Local DNS 서버를 통해서 Root DNS 서버에 요청,</li>
<li>Root DNS 서버는 TLD 주소를 응답으로 반환(com의 주소) 합니다.</li>
<li>Local DNS 서버에서 다시 TLD DNS 서버로 요청합니다.</li>
<li>TLD DNS 서버는 서브 도메인 서버의 IP 주소를 응답으로 반환(example의 주소) 합니다.</li>
<li>같은 상황을 반복해서 IP 주소를 반환 받습니다.</li>
</ol>
<p>이렇게 요청과 응답을 반복해서 IP주소를 응답 받는 방식 입니다.</p>
<h2>DNS 서버 종류</h2>
<ul>
<li>Root DNS Server
<ul>
<li>도메인 이름을 IP 주소로 변환(확인)하는 첫 번째 단계</li>
<li>요청(쿼리)를 받아 해당 도메인의 확장자(ex: .com)를 확인하고 TLD Server 보내 응답합니다.</li>
</ul>
</li>
<li>TLD DNS Server
<ul>
<li>예를 들어 <code>.com</code>으로 끝나는 모든 웹사이트 정보를 가지고 있습니다.</li>
<li>그래서 Root 서버로 응답을 받고 다음 하위 서버를 가리켜 응답합니다.</li>
</ul>
</li>
