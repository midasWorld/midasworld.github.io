<h2>GIT 모든 로그에서 특정 파일 삭제하는 방법</h2>
<pre><code class="hljs language-shell">git filter-branch --tree-filter 'rm -rf .env' HEAD
</code></pre>
<h3><strong>❗️ 어떤 문제가 발생할지 모르니... 무조건 백업 먼저 해두기!!</strong></h3>
<p>만약에 폴더에 있다면 폴더 경로까지 다 작성해주면 됩니다.
<br>
(ex: model/auth.js)</p>
<pre><code class="hljs language-shell">git filter-branch --tree-filter 'rm -rf model/auth.js' HEAD
</code></pre>
<p>그리고 커밋할 내역에 남아있다면 오류가 발생하니 미리 Commit or Stash 처리 해둬야 합니다.</p>
<h3>🤔 혹시나 폴더에 공백이 있다면? (ex: <code>/folder 1/test.sh</code>)</h3>
<p>이 경우에는 특수문자인 <code>\</code>를 사용하면 됩니다. ❗️</p>
<pre><code class="hljs language-shell">git filter-branch --tree-filter 'rm -rf folder\ 1/test.sh' HEAD
</code></pre>
<h3>🚨 혹시나 아래와 같은 오류가 발생하며 안된다면?</h3>
<pre><code class="hljs language-shell">cannot create a new backup. 
a previous backup already exists in refs/original/ force overwriting the backup with -f ...
</code></pre>
<p>위와 같은 경우에는 강제로 덮어 쓸수 있도록! <code>-f</code> 명령어를 추가해주시면 해결됩니다.</p>
<pre><code class="hljs language-shell">git filter-branch -f --tree-filter 'rm -rf .env' HEAD
