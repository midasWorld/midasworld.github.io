<h3><code>services</code>란?</h3>
<ul>
<li>실행하려는 컨테이너들을 정의하는 역할</li>
<li>이름, 이미지, 포트 매핑, 환경 변수, 볼륨 등을 포함</li>
<li>해당 정보를 가지고 컨테이너를 생성하고 관리</li>
</ul>
<h3><code>services</code> 옵션</h3>
<ul>
<li>images: 컨테이너를 생성할 때 쓰일 이미지 지정</li>
<li>build: 정의된 도커파일에서 이미지를 빌드해 서비스의 컨테이너를 생성하도록 설정</li>
<li>environment: 환경 변수 설정, docker run 명령어의 --env, -e 옵션과 동일</li>
<li>command: 컨테이너가 실행 될 때 수행할 명령어, docker run 명령어의 마지막에 붙이는 커맨드와 동일</li>
<li>depends_on: 컨테이너간의 의존성 주입, 명시된 컨테이너가 먼저 생성되고 실행</li>
<li>ports: 개방할 포트 지정, docker run 명령어의 -p 옵션과 동일 (포트 포워딩)</li>
<li>expose: 링크로 연계된 컨테이너에게만 공개할 포트 설정</li>
<li>volumes: 컨테이너에 볼륨을 마운트함</li>
<li>restart: 컨테이너가 종료될 때 재시작 정책
<ul>
<li>no: 재시작 되지 않음</li>
<li>always: 외부의 영향에 의해 종료 되었을 때 항상 재시작(수동으로 끄기 전까지)</li>
<li>on-failure: 오류가 있을 시 재시작</li>
</ul>
</li>
</ul>
<h2>도커 컴포즈 명령어</h2>
<blockquote>
<p>docker-compose -f local-infra.yml up -d</p>
</blockquote>
<ul>
<li>up: 도커 컴포즈 파일로, 컨테이너를 실행하기</li>
<li>-f: 도커 컴포즈 파일 지정하기</li>
<li>-d: 백그라운드에서 실행하기</li>
</ul>
<h3>도커 네트워크 리스트 조회</h3>
<blockquote>
<p>docker network ls</p>
</blockquote>
<ul>
<li>bridge: 도커 엔진에 의해 자동으로 생성되는 가상 네트워크, 컨테이너끼리 연결되는 기본 네트워크</li>
<li>host: 호스트 컴퓨터의 네트워크 인터페이스를 그대로 사용하는 네트워크</li>
<li>none: 네트워크를 사용하지 않는 컨테이너</li>
</ul>
<h3>도커 네트워크 생성</h3>
<blockquote>
<p>docker network create wordpress_net</p>
</blockquote>
