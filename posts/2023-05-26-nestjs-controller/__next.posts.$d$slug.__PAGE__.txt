<h2>@Param - route(routing | path) parameter</h2>
<p>params 의 기본 타입이 any로 설정되어 권장하지 않습니다.
<br> 그래서 아래와 같이 2가 방법으로 타입을 명시할 수 있습니다.</p>
<pre><code class="hljs language-typescript"><span class="hljs-comment">// 1.</span>
<span class="hljs-meta">@Delete</span>(<span class="hljs-string">'/api/v1/:userId/follows/:targetId'</span>)
<span class="hljs-title function_">deleteUserFollow</span>(<span class="hljs-params"><span class="hljs-meta">@Param</span>() <span class="hljs-attr">params</span>: { [key: <span class="hljs-built_in">string</span>]: <span class="hljs-built_in">string</span> }</span>) {
  <span class="hljs-keyword">return</span> <span class="hljs-string">`userId: <span class="hljs-subst">${params.userId}</span>, targetId: <span class="hljs-subst">${params.targetId}</span>`</span>;
}

<span class="hljs-comment">// 2.</span>
<span class="hljs-meta">@Delete</span>(<span class="hljs-string">'/api/v2/:userId/follows/:targetId'</span>)
<span class="hljs-title function_">deleteUserFollowV2</span>(<span class="hljs-params">
  <span class="hljs-meta">@Param</span>(<span class="hljs-string">'userId'</span>) <span class="hljs-attr">userId</span>: <span class="hljs-built_in">string</span>,
  <span class="hljs-meta">@Param</span>(<span class="hljs-string">'targetId'</span>) <span class="hljs-attr">targetId</span>: <span class="hljs-built_in">string</span>,
</span>) {
  <span class="hljs-keyword">return</span> <span class="hljs-string">`userId: <span class="hljs-subst">${userId}</span>, targetId: <span class="hljs-subst">${targetId}</span>`</span>;
}
</code></pre>
<p>2번째 방법을 추천합니다.
REST API 구성때 라우트 매개변수는 개수가 너무 많아지지 않게 설계하는 것을 권장하기 때문에,
코드가 많이 길어질 걱정도 없기 때문입니다.</p>
<h2>하위 도메인</h2>
<p>도메인 설정을 할때 대게는 API 도메인을 따로 두게 됩니다.
예를 들어 <code>midas.com</code> 도메인이 있다면, API 도메인을 <code>api.midas.com</code>으로 설정합니다.</p>
<p>여기서 만약에 API 도메인을 버전별로 달리 사용하고 싶다면 어떻게 해야할까요? 🤔
(ex: <code>v1.api.midas.com</code>, <code>v2.api.midas.com</code>)</p>
<p>이럴 때 <strong>하위 도메인 라우팅 기법</strong>을 사용합니다.</p>
<h3>Mac - api.localhost 도메인 설정하기</h3>
<ul>
<li>설정 환경: macOS ventura 13.3.1(a)</li>
</ul>
<p>하위 도메인 api.localhost 설정을 하기 위해서는 hosts 파일을 수정해야 합니다.</p>
<blockquote>
<p>hosts 파일이란?
<br>
host 파일은 운영 체제가 호스트 이름을 IP 주소에 매핑할 때 사용되는 파일 입니다.</p>
</blockquote>
<p>이제 수정해보겠습니다.</p>
<pre><code class="hljs language-shell">sudo vim /private/etc/hosts
