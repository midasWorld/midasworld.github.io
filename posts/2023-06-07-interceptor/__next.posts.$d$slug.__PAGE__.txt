<p>인터셉터는 AOP(Aspect Oriented Programming) 기술에 영감을 받은 유용한 기술입니다.</p>
<ul>
<li>메서드 실행 전/후 추가 로직 바인딩</li>
<li>함수에서 반환된 결과 변환</li>
<li>함수에서 발생한 예외 변환</li>
<li>기본 기능 동작 확장</li>
<li>특정 조건에 따라 함수를 완전히 재정의 (캐싱 목적)</li>
</ul>

<br>그 로깅을 만들어보면 아래와 같습니다.</p>

<h3>CallHandler</h3>
<p>이 인터페이스는 인터셉터의 특정 지점에서 Route handler 메서드를 호출하는데 사용할 수 있는 handle() 메서드를 구현합니다.
<br><strong>만약에 handle() 메서드를 호출하지 않으면 Route handler 메서드 실행되지 않습니다.</strong>
<br>이를 통해 request/response 스트림을 효과적으로 <strong>wraps</strong> 할 수 있으며 전/후 처리를 커스텀 할 수 있습니다.</p>
<p>이 handle() 메서드는 Observable 를 반환하기 때문에, 응답을 추가로 조작할 수 있는데요.
<br>Observable 이라는 Rxjs 연산자는 무엇일까요? 🤔</p>
<h3>Rxjs 란?</h3>
<ul>
<li>해당 유튜브 세션 영상 : <a href="https://www.youtube.com/watch?v=VH1GTGIMHQw&#x26;t=5827s">https://www.youtube.com/watch?v=VH1GTGIMHQw&#x26;t=5827s</a></li>
</ul>
<p>RxJS 는 이벤트 스트림을 다루는 라이브러리로,
<br>이를 operator를 이용해 변환할 수 있습니다.</p>
<blockquote>
<p>RxJS == 함수형 &#x26; 이벤트 &#x26; 비동기</p>
</blockquote>
<h3>Observable 이란?</h3>
<ul>
<li>이벤트가 흐르는 스트림</li>
<li>lazy 한 실행 (누군가 subscribe 를 해야 실행 됨)</li>
<li>observer 가 observable 을 subscribe 하면서 next, error, complete 키워드를 사용해 이벤트 처리</li>
<li>위 코드(logging.inteceptor.ts) 처럼 pipe() operator 를 사용해서 스트림 내부에서 변환 처리 가능</li>
</ul>
<blockquote>
<p>💡 observer 는 observable 을 구독하는 대상 > observable 에 들어오는 event 를 처리합니다.</p>
</blockquote>
<blockquote>
<p>💡 operator 는 각 이벤트 연산이 가능한 pure function 입니다. (tap(), filter(), min(), max() 등이 있음)</p>
</blockquote>

<h2>Interceptor vs Middleware 차이</h2>
<ul>
<li>도움된 글 링크 : <a href="https://blog-ko.superb-ai.com/nestjs-interceptor-and-lifecycle/">https://blog-ko.superb-ai.com/nestjs-interceptor-and-lifecycle/</a></li>
</ul>
<p>위의 글을 보면...</p>
<h3>Middleware</h3>
<ul>
<li>request, response, next 인자를 받아 req, res가 HTTP 위에서 동작하게 설계</li>
<li>HTTP 통신이 아니면 사용이 불가능</li>
</ul>
<h3>Interceptor</h3>
<ul>
<li>HTTP(request, response, next) 외에도 매개변수로 <strong>Excuetion Context</strong> 라는 helper class를 받음</li>
<li>HTTP 외에도 WebSocket, GraphQL, RPC(Remote procedure call) 위에서도 동작!</li>
