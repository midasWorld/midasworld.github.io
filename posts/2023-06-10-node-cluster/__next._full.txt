14:T38f8,<p>NodeJS 의 장점이자 단점이 바로 Non-blocking I/O 와 단일 스레드 입니다.</p>
<p>싱글 스레드로 하나의 코어에서 실행되기 때문에,
<br>과부하가 된다면 서버가 다운되는 현상을 겪을 수도 있습니다.</p>
<blockquote>
<p>🤔 이를 해결하기 위한 방법은?</p>
</blockquote>
<ol>
<li>클러스터링 : <a href="https://nodejs.org/api/cluster.html">https://nodejs.org/api/cluster.html</a>
<br> 여러 프로세스를 생성하여 요청을 병렬로 처리하는 방식입니다.
<br> 각 프로세스는 독립적으로 작동하며, 요청 부하를 분산시킴으로써 동시성을 향상시킵니다.</li>
<li>스레드풀 사용 : <a href="https://nodejs.org/api/worker_threads.html">https://nodejs.org/api/worker_threads.html</a>
<br> worker-thread 를 사용하여 CPU 집약적인 작업을 병렬로 처리할 수 있습니다.
<br> worker-thread 는 Node 의 메인 스레드와 별도로 동작하며,
<br> 비동기적으로 작업을 처리하여 동시성을 개선합니다.</li>
</ol>
<h3>🛠️ 과부하 테스트 툴</h3>
<ul>
<li>Artillery : <a href="https://github.com/artilleryio/artillery/tree/main/packages/artillery#readme">https://github.com/artilleryio/artillery/tree/main/packages/artillery#readme</a></li>
</ul>
<p>위의 툴은 지정한 수의 요청을 보낼 수 있는 과부화 툴 입니다. 😁</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_"># </span><span class="bash">설치</span>
npm install -g artillery
<span class="hljs-meta prompt_">
# </span><span class="bash">사용</span>
artillery quick -c 2 -n 4000 http://localhost:8000
</code></pre>
<p>여기서 사용 부분을 보시면,
<br><code>-c</code>는 가상 사용자의 수, <code>-n</code>은 보낼 요청의 수 입니다.
<br>고로 2명의 가상 사용자가 4,000번의 요청을 보내는 테스트 입니다.</p>
<p>공식 문서를 보시면 yml 파일을 작성하여 시나리오까지 만들 수 있습니다. 😁</p>

<br>클러스터링을 적용하면 완벽하게 나누어서 동작할 것 같지만 꼭 그렇지도 않다는 것을 확인할 수 있습니다.</p>
<p>따로 지정하지 않는다면 기본적으로 라운드 로빈 방식으로 동작하게 되는데요,
<br>라운드 로빈 방식은 순차적으로 하나의 작업을 배분하는 방식입니다.</p>
<p>그래서 각각에 사용자의 담당을 순차적으로 맡겨서 실행하게 됩니다.</p>
<p>트래픽이 분산되는 것은 맞지만, 모든 요청에 대해서 모든 작업이 나누어 실행되지는 않습니다.
<br>이를 위해서는 로드밸런싱이 필요하다는 것을 알 수 있습니다.</p>
<h2>🪄 PM2</h2>
<p>위와 같이 코드를 작성하지 않고도 PM2 라는 툴만 이용하면 간단하게 클러스터 사용 가능합니다.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_"># </span><span class="bash">설치</span>
npm install -g pm2
<span class="hljs-meta prompt_">
# </span><span class="bash">사용법</span>
pm2 start &#x3C;file> -i &#x3C;n>
pm2 start ./app.js -i max
</code></pre>
<p>위와 같이 max 라고 지정하게 되면 최대 CPU 개수로 실행 됩니다.</p>
