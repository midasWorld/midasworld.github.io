14:T43f5,<blockquote>
<p>✨ 데이터베이스 운영에서 가장 중요한 두 가지 요소는 확장성과 가용성 입니다.</p>
</blockquote>
<p>이 두 요소를 위해 가장 일반적으로 사용되는 기술이 '복제(Replication)' 입니다.</p>
<h2>Replication 사용 이유</h2>
<img src="../../assets/img/docs/database/mysql_replication.png" />
<ol>
<li>
<p>스케일 아웃 (Scale-out)</p>
<ul>
<li>대량의 트래픽을 처리하기 위해 DB 서버의 개수를 늘려 실행 되는 쿼리를 분산시키기 위해 사용됩니다.</li>
<li>스케일 아웃은 스케일 업 방식보다 갑자기 늘어나는 트래픽을 대응하는 데 훨씬 더 유연한 구조입니다.</li>
</ul>
</li>
<li>
<p>데이터 백업</p>
<ul>
<li>DB 백업은 필수 입니다. (많이 하면 할수록 좋습니다. 👍)</li>
<li>하지만 사용중인 DB를 백업하게 되면 DBMS가 서버의 자원을 공유해서 사용하기 때문에,백업으로 인해 DBMS에서 실행 중인 쿼리들이 영향을 받을 수 있으며, 처리속도가 늦어져 문제가 발생할 수 있습니다.</li>
<li>이러한 문제를 방지하기 위해 레플리카 서버를 구축하고, 백업은 여기서 실행 됩니다.</li>
<li>📌 <strong>백업용 레플리카 서버는 소스 서버가 문제가 생겼을 때를 대비한 서버의 역할</strong></li>
</ul>
</li>
<li>
<p>데이터 분석</p>
<ul>
<li>많은 기업에서 데이터 분석을 목적으로 분석용 쿼리를 사용하게 됩니다.</li>
<li>이러한 분석용 쿼리는 대량의 데이터를 집계 연산을 포함해서 하기 때문에, 복잡하고 무겁습니다.</li>
<li>서버의 많은 리소스를 사용하기 때문에 여분의 레플리카 서버를 구축해 분석용 쿼리 전용으로 실행하는 것이 좋습니다.</li>
</ul>
</li>
<li>
<p>데이터 지리적 분산</p>
<ul>
<li>DB 서버와 App 서버가 서로 떨어져 있는 경우, 두 서버 간 통신 시간은 거리만큼 늘어납니다.</li>
<li>이를 위해 DB 서버를 복제해 App 서버가 위치한 곳에 레플리카 서버를 구축해 응답 속도를 개선할 수 있습니다.</li>
</ul>
</li>
</ol>

<h3>[master] master.cnf 파일 > etc/mysql/my.cnf</h3>
<pre><code class="hljs language-yaml">[<span class="hljs-string">mysqld</span>]
<span class="hljs-string">server_id=10</span>
<span class="hljs-string">log_bin=mysql-bin</span>
<span class="hljs-string">default_authentication_plugin=mysql_native_password</span>
<span class="hljs-string">binlog_do_db=</span>
</code></pre>
<ul>
<li>server_id : 서버의 고유 식별자 입니다.
<br> 복제 DB 와는 다르게 각각 설정해줘야 합니다.</li>
<li>log_bin : 바이너리 로그의 이름을 지정할 수 있습니다.</li>
<li>binlog_do_db : 복제할 데이터베이스를 지정합니다.
<br> (따로 지정해주지 않으면 mydatabase 라는 기본값이 들어갔습니다. 😓)</li>
<li>default_authentication_plugin : 기본 인증 플러그인을 지정하는데 사용됩니다.
<br>mysql_native_password 는 전통적인 비밀번호 인증 방식을 사용하는 플러그인 입니다.</li>
</ul>

<h3>[slave] slave.cnf 파일 > etc/mysql/my.cnf</h3>
<pre><code class="hljs language-yaml">[<span class="hljs-string">mysqld</span>]
<span class="hljs-string">server_id=11</span>
<span class="hljs-string">relay_log=/var/lib/mysql/mysql-relay-bin</span>
<span class="hljs-string">log_slave_updates</span>
<span class="hljs-string">read_only</span>
<span class="hljs-string">default_authentication_plugin=mysql_native_password</span>
</code></pre>
<ul>
<li>relay_log : 릴레이 로그 파일명을 지정할 수 있습니다.
<br>릴레이 로그는 마스터 서버의 바이너리 로그에서 읽어온 이벤트(트랜잭션) 정보가 저장됩니다.
<br>여기에 저장된 이벤트들이 Replication SQL 스레드에 의해 레플리카 서버에 적용됩니다.</li>
<li>log_slave_updates : 레플리카 서버에서 수신한 변경 작업도 이벤트 로그에 적용할 지 여부를 지정할 수 있습니다.</li>
<li>read_only : 레플리카 서버를 읽기 전용으로 설정하는데 사용 됩니다.</li>
</ul>
