<h2>1. Claude Code란?</h2>
<p>Claude Code는 Anthropic이 만든 <strong>터미널 기반 AI 코딩 에이전트</strong>다. 단순한 챗봇이 아니라 실제 파일을 읽고, 편집하고, 명령어를 실행하며 코드베이스 전체와 상호작용한다.</p>

<p>Cursor는 에디터 안에서 코드를 자동 완성하거나 채팅하는 느낌이라면, Claude Code는 <strong>시니어 개발자를 고용해서 터미널 앞에 앉혀 놓은 것</strong>에 가깝다. 스스로 파일 구조를 파악하고, 여러 파일을 동시에 수정하며, 빌드가 실패하면 오류를 읽고 다시 고친다.</p>
<h3>설치 방법</h3>
<pre><code class="hljs language-bash"><span class="hljs-comment"># Node.js 18+ 필요</span>
npm install -g @anthropic-ai/claude-code

<span class="hljs-comment"># 설치 확인</span>
claude --version

<span class="hljs-comment"># 시작</span>
claude
</code></pre>
<p>처음 실행하면 Anthropic API 키를 물어본다. <a href="https://console.anthropic.com">console.anthropic.com</a>에서 발급받아 입력한다.</p>

<h2>2. 토큰 절약 방법 (핵심)</h2>
<p>Claude Code의 비용은 토큰 사용량에 비례한다. 잘 쓰면 같은 예산으로 두 배 이상 효율을 낼 수 있다.</p>
<h3>CLAUDE.md 활용</h3>
<p>프로젝트 루트에 <code>CLAUDE.md</code> 파일을 두면 <strong>매 대화마다 자동으로 컨텍스트가 주입</strong>된다. 프로젝트 구조, 빌드 명령어, 코드 컨벤션을 한 번 써두면 매번 설명할 필요가 없어진다.</p>
<pre><code class="hljs language-markdown"><span class="hljs-section"># CLAUDE.md</span>

<span class="hljs-section">## 프로젝트 개요</span>
Next.js 14 + TypeScript 기술 블로그. <span class="hljs-code">`output: 'export'`</span>로 정적 빌드.

<span class="hljs-section">## 주요 명령어</span>
<span class="hljs-bullet">-</span> <span class="hljs-code">`npm run dev`</span> - 개발 서버
<span class="hljs-bullet">-</span> <span class="hljs-code">`npm run build`</span> - 프로덕션 빌드
<span class="hljs-bullet">-</span> <span class="hljs-code">`npm test`</span> - 테스트 실행

<span class="hljs-section">## 디렉토리 구조</span>
<span class="hljs-bullet">-</span> <span class="hljs-code">`app/`</span> - Next.js App Router 페이지
<span class="hljs-bullet">-</span> <span class="hljs-code">`components/`</span> - 재사용 컴포넌트
<span class="hljs-bullet">-</span> <span class="hljs-code">`lib/posts.ts`</span> - 서버 전용 (fs 사용, 클라이언트에서 import 금지)
<span class="hljs-bullet">-</span> <span class="hljs-code">`lib/types.ts`</span> - 타입 정의 (클라이언트 안전)

<span class="hljs-section">## 코드 컨벤션</span>
<span class="hljs-bullet">-</span> 함수형 컴포넌트 + TypeScript strict 모드
<span class="hljs-bullet">-</span> 불변성 유지: 객체 직접 변경 금지, 새 객체 반환
<span class="hljs-bullet">-</span> 파일당 최대 400줄

<span class="hljs-section">## 절대 건드리면 안 되는 파일</span>
<span class="hljs-bullet">-</span> <span class="hljs-code">`.env.local`</span> - 환경 변수
<span class="hljs-bullet">-</span> <span class="hljs-code">`content/private/`</span> - 암호화된 비공개 콘텐츠
</code></pre>
<p>이렇게 해두면 Claude가 프로젝트를 처음 보더라도 맥락을 바로 파악한다. <strong>반복 설명에 쓰이는 토큰이 크게 줄어든다.</strong></p>
<h3><code>/compact</code> 명령</h3>
<p>대화가 길어지면 컨텍스트 창이 꽉 차기 시작한다. 이때 <code>/compact</code>를 치면 <strong>현재까지의 대화를 요약</strong>하여 토큰을 절약하면서 대화를 이어갈 수 있다.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_">> </span><span class="bash">/compact</span>
</code></pre>
<h3><code>/clear</code> vs <code>/compact</code> 차이</h3>
<table>
<thead>
<tr>
<th>명령어</th>
<th>동작</th>
<th>사용 시점</th>
</tr>
</thead>
<tbody>
<tr>
<td><code>/clear</code></td>
<td>대화 기록 완전 초기화</td>
<td>완전히 새로운 작업 시작</td>
</tr>
<tr>
<td><code>/compact</code></td>
<td>대화를 요약하여 압축</td>
<td>같은 작업을 계속하되 토큰 절약</td>
</tr>
</tbody>
</table>
<p>긴 리팩토링 세션 중간에는 <code>/compact</code>, 완전히 다른 작업으로 넘어갈 때는 <code>/clear</code>를 쓴다.</p>
<h3>구체적으로 질문하기</h3>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 나쁜 예 - 모호한 질문 → 긴 답변 → 토큰 낭비</span>
<span class="hljs-string">"이 프로젝트에서 성능 문제 찾아줘"</span>

<span class="hljs-comment"># 좋은 예 - 범위를 좁힌 질문</span>
<span class="hljs-string">"src/lib/posts.ts의 getAllPosts 함수가 매 요청마다 파일을 전부 읽는데, 캐싱 로직 추가해줘"</span>
</code></pre>
<h3>파일 직접 지정</h3>
<p>Claude Code는 필요하면 알아서 파일을 탐색하지만, 직접 지정하면 탐색 비용이 줄어든다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 비효율적</span>
<span class="hljs-string">"로그인 버그 찾아줘"</span>

<span class="hljs-comment"># 효율적</span>
<span class="hljs-string">"src/auth/login.ts 38번째 줄 확인해줘. userId가 null일 때 500 에러 발생함"</span>
</code></pre>
<h3><code>--model</code> 옵션으로 저렴한 모델 활용</h3>
<p>단순한 작업에는 Haiku 모델을 쓰면 비용이 크게 줄어든다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 단순 포맷팅, 타입 수정 등</span>
claude --model claude-haiku-4-5

<span class="hljs-comment"># 복잡한 설계, 아키텍처 결정</span>
claude --model claude-opus-4-5

<span class="hljs-comment"># 기본 개발 작업 (기본값)</span>
claude --model claude-sonnet-4-6
</code></pre>
<p>작업 복잡도에 따라 모델을 골라 쓰는 것이 가장 경제적이다.</p>
<h3>불필요한 확인 줄이기</h3>
<p>신뢰할 수 있는 환경에서는 매번 나오는 확인 프롬프트를 줄일 수 있다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 주의: 신뢰된 로컬 환경에서만 사용</span>
claude --dangerously-skip-permissions
</code></pre>
<p>프로덕션 서버나 중요한 환경에서는 절대 사용하지 말 것.</p>

<h2>3. CLAUDE.md 작성 요령</h2>
<p>잘 쓴 CLAUDE.md 하나가 수십 번의 반복 설명을 대체한다.</p>
<h3>포함해야 할 항목</h3>
<ol>
<li><strong>프로젝트 한 줄 요약</strong> - 무엇을 하는 프로젝트인지</li>
<li><strong>자주 쓰는 명령어</strong> - 빌드, 테스트, 실행, 배포</li>
<li><strong>디렉토리 구조 설명</strong> - 핵심 디렉토리의 역할</li>
<li><strong>코드 스타일 규칙</strong> - 네이밍, 패턴, 금지 사항</li>
<li><strong>절대 수정 금지 파일</strong> - 실수로 건드리면 안 되는 것들</li>
<li><strong>환경 변수 설명</strong> - 어떤 값이 필요한지 (실제 값 말고 키 이름만)</li>
</ol>
<h3>실전 CLAUDE.md 예시 (백엔드 API 서버)</h3>
<pre><code class="hljs language-markdown"><span class="hljs-section"># CLAUDE.md</span>

<h2>4. 추천 워크플로우</h2>
<h3>Plan → Code → Review 패턴</h3>
<p>큰 작업을 시작할 때는 바로 코드 작성을 요청하지 말고, 먼저 계획을 세우게 한다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># Step 1: 계획 요청</span>
<span class="hljs-string">"users 모듈에 소셜 로그인(Google, GitHub) 기능을 추가하려고 해.
어떤 파일을 어떻게 수정해야 할지 계획만 세워줘. 코드는 아직 짜지 마."</span>

<span class="hljs-comment"># Step 2: 계획 검토 후 승인</span>
<span class="hljs-string">"3번 방법으로 진행해줘. 단, JWT 갱신 로직은 기존 auth.service.ts를 재사용해."</span>

<span class="hljs-comment"># Step 3: 단계별 구현</span>
<span class="hljs-string">"auth/social 디렉토리 구조 먼저 만들어줘."</span>
</code></pre>
<p>이렇게 하면 방향이 잘못된 채로 수백 줄을 짜는 일을 막을 수 있다.</p>
<h3><code>/plan</code> 모드 활용</h3>
<p><code>/plan</code> 명령어를 쓰면 코드 수정 없이 계획만 세우는 모드로 전환된다.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_">> </span><span class="bash">/plan</span>
계획 모드 진입. 파일을 수정하지 않고 분석과 계획만 제시합니다.
<span class="hljs-meta prompt_">
> </span><span class="bash">인증 시스템을 JWT에서 세션 기반으로 마이그레이션하는 계획 세워줘</span>
</code></pre>
<p>계획을 보고 마음에 들면 <code>/plan</code> 모드를 끄고 구현을 요청한다.</p>
<h3>단계별 작업 요청</h3>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 나쁜 예 - 한 번에 너무 많이</span>
<span class="hljs-string">"결제 모듈 전체 만들어줘. Stripe 연동하고, 웹훅 처리하고, 환불 기능도 추가해."</span>

<span class="hljs-comment"># 좋은 예 - 단계별로</span>
1. <span class="hljs-string">"Stripe SDK 설치하고 payment.module.ts 기본 구조 잡아줘"</span>
2. <span class="hljs-string">"결제 생성 엔드포인트 먼저 구현해줘"</span>
3. <span class="hljs-string">"웹훅 핸들러 추가해줘"</span>
4. <span class="hljs-string">"환불 로직 마지막으로 추가해줘"</span>
</code></pre>
<h3>체크포인트 커밋</h3>
<p>큰 작업 중간에는 자주 커밋해두는 것이 중요하다. Claude가 잘못된 방향으로 가도 롤백이 쉽다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 작업 중간중간</span>
git add -A &#x26;&#x26; git commit -m <span class="hljs-string">"wip: 소셜 로그인 기본 구조 추가"</span>

<span class="hljs-comment"># 완성 후</span>
git commit -m <span class="hljs-string">"feat: Google/GitHub 소셜 로그인 구현"</span>
</code></pre>
<p>CLAUDE.md에 이런 규칙을 명시해두면 Claude도 알아서 커밋을 제안한다.</p>

<h2>6. MCP (Model Context Protocol) 서버 활용</h2>
<p>MCP는 Claude에게 외부 도구나 API를 연결하는 프로토콜이다. 일종의 플러그인 시스템으로, MCP 서버를 통해 Claude가 GitHub, DB, 웹 등에 직접 접근할 수 있게 된다.</p>
<h3>추천 MCP 서버</h3>
<p><strong>파일 시스템 접근</strong></p>
<pre><code class="hljs language-bash">@modelcontextprotocol/server-filesystem
</code></pre>
<p>특정 디렉토리에만 접근을 허용하도록 제한할 수 있어서 보안상 안전하다.</p>
<p><strong>GitHub 연동</strong></p>
<pre><code class="hljs language-bash">@modelcontextprotocol/server-github
</code></pre>
<p>PR 생성, 이슈 조회, 코드 리뷰 댓글 달기 등을 Claude가 직접 처리한다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 활용 예시</span>
<span class="hljs-string">"현재 브랜치의 변경사항을 분석해서 PR 설명 작성하고 GitHub에 PR 올려줘"</span>
</code></pre>
<p><strong>PostgreSQL 연동</strong></p>
<pre><code class="hljs language-bash">@modelcontextprotocol/server-postgres
</code></pre>
<p>스키마를 분석하거나 쿼리 최적화를 요청할 때 DB에 직접 접근하게 할 수 있다.</p>
<p><strong>웹 검색</strong></p>
<pre><code class="hljs language-sql"><span class="hljs-variable">@modelcontextprotocol</span><span class="hljs-operator">/</span>server<span class="hljs-operator">-</span>brave<span class="hljs-operator">-</span><span class="hljs-keyword">search</span>
</code></pre>
<p>Claude가 최신 문서나 패키지 정보를 직접 검색할 수 있게 된다.</p>
<p><strong>브라우저 자동화</strong></p>
<pre><code class="hljs language-bash">@modelcontextprotocol/server-puppeteer
</code></pre>
<p>E2E 테스트나 웹 스크래핑을 자동화할 때 유용하다.</p>

<h2>8. 권한 설정 꿀팁</h2>
<p>Claude Code는 파일 수정, 명령어 실행 등 강력한 기능을 가지므로 권한 설정이 중요하다.</p>
<h3>도구별 권한 제어</h3>
<pre><code class="hljs language-json"><span class="hljs-punctuation">{</span>
  <span class="hljs-attr">"allowedTools"</span><span class="hljs-punctuation">:</span> <span class="hljs-punctuation">[</span>
    <span class="hljs-string">"Read"</span><span class="hljs-punctuation">,</span>
    <span class="hljs-string">"Edit"</span><span class="hljs-punctuation">,</span>
    <span class="hljs-string">"Write"</span><span class="hljs-punctuation">,</span>
    <span class="hljs-string">"Glob"</span><span class="hljs-punctuation">,</span>
    <span class="hljs-string">"Grep"</span><span class="hljs-punctuation">,</span>
    <span class="hljs-string">"Bash"</span>
  <span class="hljs-punctuation">]</span>
<span class="hljs-punctuation">}</span>
</code></pre>
<p><code>Bash</code>를 허용 목록에서 빼면 터미널 명령어 실행을 막을 수 있다. 코드 리뷰만 시킬 때는 <code>Read</code>, <code>Glob</code>, <code>Grep</code>만 허용하는 것도 방법이다.</p>
<h3>일회성 도구 허용</h3>
<p>세션마다 설정 파일을 바꾸는 대신, 실행 시 플래그로 일회성 허용이 가능하다.</p>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 이 세션에서만 Bash 도구 허용</span>
claude --allowedTools <span class="hljs-string">"Read,Edit,Write,Bash"</span>

<span class="hljs-comment"># 이 세션에서만 읽기 전용</span>
claude --allowedTools <span class="hljs-string">"Read,Glob,Grep"</span>
</code></pre>
<h3>위험한 명령어 실행 전 항상 확인</h3>
<p>Claude가 <code>rm -rf</code>, <code>DROP TABLE</code> 같은 위험한 명령어를 실행하려 할 때는 반드시 멈추고 내용을 확인한다. Claude Code는 기본적으로 위험한 명령어 실행 전에 확인을 요청하지만, 습관적으로 직접 검토하는 것이 중요하다.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_"># </span><span class="bash">Claude가 이런 명령어를 제안하면 반드시 내용 확인</span>
<span class="hljs-meta prompt_">> </span><span class="bash">다음 명령어를 실행하겠습니다: <span class="hljs-built_in">rm</span> -rf node_modules &#x26;&#x26; npm install</span>
<span class="hljs-meta prompt_">> </span><span class="bash">실행할까요? [y/N]</span>
</code></pre>

<h3>생성된 코드는 반드시 직접 검토</h3>
<p>Claude가 생성한 코드는 항상 훑어본다. 특히:</p>
<ul>
<li>보안 관련 코드 (인증, 권한, 암호화)</li>
<li>DB 쿼리 (특히 DELETE, UPDATE)</li>
<li>외부 API 호출</li>
<li>파일 시스템 작업</li>
</ul>
<p>Claude는 빠르고 유능하지만 실수를 한다. 최종 판단은 개발자가 한다.</p>
