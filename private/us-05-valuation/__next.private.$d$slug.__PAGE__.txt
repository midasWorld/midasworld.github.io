<h2>1. PER (Price-to-Earnings Ratio) 완전 정리</h2>
<h3>1-1. 기본 계산법과 의미</h3>
<pre><code class="hljs language-ini"><span class="hljs-attr">PER</span> = 주가 ÷ 주당순이익(EPS)

예시) Apple 2024년
  주가: $185
  EPS: $6.43
  <span class="hljs-attr">PER</span> = <span class="hljs-number">185</span> ÷ <span class="hljs-number">6.43</span> ≈ <span class="hljs-number">28.8</span>배

의미: "현재 이익 기준으로 주가를 회수하는 데 28.8년 소요"
</code></pre>

<h3>1-3. Trailing PER vs Forward PER</h3>
<table>
<thead>
<tr>
<th>구분</th>
<th>정의</th>
<th>장점</th>
<th>단점</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Trailing PER</strong></td>
<td>과거 12개월 실제 EPS 기준</td>
<td>확정된 수치, 신뢰성 높음</td>
<td>과거 데이터, 미래 반영 불가</td>
</tr>
<tr>
<td><strong>Forward PER</strong></td>
<td>향후 12개월 예상 EPS 기준</td>
<td>미래 성장성 반영</td>
<td>애널리스트 추정, 빗나갈 수 있음</td>
</tr>
</tbody>
</table>
<pre><code class="hljs language-sql">[<span class="hljs-keyword">Trailing</span> vs Forward <span class="hljs-keyword">PER</span> 비교 실전 예시]

Nvidia (NVDA) <span class="hljs-number">2024</span>년 초
  주가: $<span class="hljs-number">490</span>
  최근 <span class="hljs-number">12</span>개월 EPS: $<span class="hljs-number">12.96</span>
  <span class="hljs-keyword">Trailing</span> <span class="hljs-keyword">PER</span> <span class="hljs-operator">=</span> <span class="hljs-number">490</span> ÷ <span class="hljs-number">12.96</span> ≈ <span class="hljs-number">37.8</span>배

  향후 <span class="hljs-number">12</span>개월 예상 EPS: $<span class="hljs-number">24.58</span> (AI 수요 급증 반영)
  Forward <span class="hljs-keyword">PER</span> <span class="hljs-operator">=</span> <span class="hljs-number">490</span> ÷ <span class="hljs-number">24.58</span> ≈ <span class="hljs-number">19.9</span>배

→ <span class="hljs-keyword">Trailing</span> <span class="hljs-keyword">PER</span> <span class="hljs-number">37.8</span>배 <span class="hljs-operator">=</span> 비싸 보임
→ Forward <span class="hljs-keyword">PER</span> <span class="hljs-number">19.9</span>배 <span class="hljs-operator">=</span> 성장 감안하면 합리적

빠르게 성장하는 기업일수록
<span class="hljs-keyword">Trailing</span> <span class="hljs-keyword">PER</span>과 Forward <span class="hljs-keyword">PER</span> 사이 차이가 크다.
</code></pre>

<h2>2. PEG Ratio — "성장을 감안한 PER"</h2>
<h3>2-1. PEG 계산법</h3>
<pre><code class="hljs language-yaml"><span class="hljs-string">PEG</span> <span class="hljs-string">=</span> <span class="hljs-string">PER</span> <span class="hljs-string">÷</span> <span class="hljs-string">연간</span> <span class="hljs-string">EPS</span> <span class="hljs-string">성장률(%)</span>

<span class="hljs-string">PEG</span> <span class="hljs-string">&#x3C;</span> <span class="hljs-attr">1.0:</span> <span class="hljs-string">저평가</span> <span class="hljs-string">(성장</span> <span class="hljs-string">대비</span> <span class="hljs-string">싸다)</span>
<span class="hljs-string">PEG</span> <span class="hljs-number">1.0</span> <span class="hljs-string">~</span> <span class="hljs-attr">1.5:</span> <span class="hljs-string">적정</span> <span class="hljs-string">평가</span>
<span class="hljs-string">PEG</span> <span class="hljs-number">1.5</span> <span class="hljs-string">~</span> <span class="hljs-attr">2.5:</span> <span class="hljs-string">약간</span> <span class="hljs-string">고평가</span>
<span class="hljs-string">PEG</span> <span class="hljs-string">></span> <span class="hljs-attr">2.5:</span> <span class="hljs-string">고평가</span> <span class="hljs-string">주의</span>
</code></pre>
<h3>2-2. 왜 성장주에 PEG가 필요한가</h3>
<pre><code class="hljs language-ini">┌──────────────────────────────────────────────────────────┐
│                    두 기업 비교                           │
│                                                          │
│  기업 A (전통 제조업)      기업 B (고성장 테크)           │
│  PER: 10배                PER: 40배                     │
│  연간 성장률: 5%           연간 성장률: 35%              │
│  <span class="hljs-attr">PEG</span> = <span class="hljs-number">10</span>÷<span class="hljs-number">5</span> = <span class="hljs-number">2.0</span>        PEG = <span class="hljs-number">40</span>÷<span class="hljs-number">35</span> = <span class="hljs-number">1.14</span>            │
│                                                          │
│  → PER만 보면 A가 훨씬 싸 보인다                         │
│  → PEG로 보면 B가 성장 대비 더 합리적 가격               │
│                                                          │
│  10년 후 예상 이익:                                       │
│  A: $100 × (1.05)^<span class="hljs-attr">10</span> = <span class="hljs-variable">$163</span> (<span class="hljs-number">1.6</span>배)                    │
│  B: $100 × (1.35)^<span class="hljs-attr">10</span> = <span class="hljs-variable">$2</span>,<span class="hljs-number">058</span> (<span class="hljs-number">20.6</span>배)                 │
└──────────────────────────────────────────────────────────┘
</code></pre>

<h2>3. EV/EBITDA — 부채까지 고려한 기업 가치</h2>
<h3>3-1. EV/EBITDA가 필요한 이유</h3>
<pre><code class="hljs language-bash">[PER의 한계 — 부채를 무시한다]

기업 A: 시총 <span class="hljs-variable">$100</span>억, 순이익 <span class="hljs-variable">$10</span>억, 부채 <span class="hljs-variable">$0</span>
기업 B: 시총 <span class="hljs-variable">$100</span>억, 순이익 <span class="hljs-variable">$10</span>억, 부채 <span class="hljs-variable">$80</span>억

PER로 보면: 둘 다 10배로 동일 → 비교 불가
EV/EBITDA로 보면:
  기업 A: EV = <span class="hljs-variable">$100</span>억, EBITDA = <span class="hljs-variable">$12</span>억, EV/EBITDA = 8.3배
  기업 B: EV = <span class="hljs-variable">$180</span>억, EBITDA = <span class="hljs-variable">$12</span>억, EV/EBITDA = 15배
         (부채 <span class="hljs-variable">$80</span>억 포함)
  → 기업 B가 훨씬 비싸다는 것이 드러남
</code></pre>
<h3>3-2. EV/EBITDA 계산</h3>
<pre><code class="hljs language-ini">EV (기업가치) = 시가총액 + 순부채(부채 - 현금)

<span class="hljs-attr">EBITDA</span> = 이자·세금·감가상각 차감 전 이익
       = 영업이익 + 감가상각비(D&#x26;A)

EV/<span class="hljs-attr">EBITDA</span> = EV ÷ EBITDA
</code></pre>
<h3>3-3. Amazon — EV/EBITDA가 필수인 이유</h3>
<pre><code class="hljs language-markdown">Amazon (AMZN) 2022년 사례
────────────────────────────────────────
Trailing PER: 적자에 가까움 (또는 수백 배)
EV/EBITDA: 20~25배

이유:
<span class="hljs-bullet">  -</span> Amazon은 물류·AWS 인프라에 천문학적 CAPEX 투자
<span class="hljs-bullet">  -</span> 감가상각비가 순이익을 압도적으로 잠식
<span class="hljs-bullet">  -</span> 실제 영업 현금창출력(EBITDA)은 탄탄하지만
<span class="hljs-code">    PER만 보면 상장 불가능해 보이는 수준
</span>
결론:
  Amazon 같은 CAPEX 집약적 기업은
  PER이 아닌 EV/EBITDA 또는 FCF(자유현금흐름)로 평가해야
  실제 가치를 파악할 수 있다
────────────────────────────────────────
</code></pre>

<h2>4. PSR (Price-to-Sales Ratio) — 적자 기업의 구원투수</h2>
<h3>4-1. 언제 PSR을 쓰는가</h3>
<pre><code class="hljs language-ini"><span class="hljs-attr">PSR</span> = 시가총액 ÷ 연간 매출액

사용 조건:
  ✓ 이익이 없는(적자) 초기 성장 기업
  ✓ 이익은 있지만 극히 적어 PER이 수백 배인 기업
  ✓ 사업 초기라 이익보다 매출 성장이 더 중요한 기업

예시: Airbnb 상장 초기 (2020~2021)
  연간 매출: $3.4B (회복 중)
  순손실: -$1.6B
  시가총액: $85B
  PER → 계산 불가 (적자)
  <span class="hljs-attr">PSR</span> = <span class="hljs-number">85</span>B ÷ <span class="hljs-number">3.4</span>B = <span class="hljs-number">25</span>배
  → 고성장 플랫폼 기업 PSR 20~30배 범위 내 → 비교 가능
</code></pre>

<h3>4-3. PSR의 함정 — 마진과 함께 봐야 한다</h3>
<pre><code class="hljs language-ini"><span class="hljs-section">[PSR 함정 사례]</span>

회사 A: PSR 3배, 매출총이익률(Gross Margin) 70%
  → 고마진 SaaS 기업 → 3배 <span class="hljs-attr">PSR</span> = 저렴
  → 이익으로 전환 가능성 높음

회사 B: PSR 3배, 매출총이익률 10%
  → 저마진 리테일 기업 → 3배 <span class="hljs-attr">PSR</span> = 매우 비쌈
  → 이익 창출 가능성 낮음

<span class="hljs-section">[PSR + Gross Margin 조합 분석]</span>

  SaaS 기업 (Gross Margin 70%+)
  PSR ≤ 10배 → 저평가 가능성
  PSR 15~25배 → 적정 (고성장 전제)
  PSR 30배+ → 고평가 주의

  리테일/에너지 (Gross Margin 20% 미만)
  PSR ≤ 0.5배 → 저평가 가능성
  PSR 1배+ → 고평가
</code></pre>
<blockquote>
<p><strong>⚠️ 주의사항</strong>: PSR은 수익성을 무시한다. 매출이 아무리 높아도 영원히 적자인 기업은 결국 자금이 소진된다. PSR은 반드시 Gross Margin, 현금 소진율(Burn Rate)과 함께 봐야 한다.</p>
</blockquote>

<h2>5. P/B (Price-to-Book) — 자산 가치 대비 주가</h2>
<h3>5-1. P/B 계산과 의미</h3>
<pre><code class="hljs language-css"><span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> = 주가 ÷ 주당 순자산(BPS)

또는

<span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> = 시가총액 ÷ 자기자본(Book Value)

<span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">1</span>배 = 주가가 순자산과 동일
<span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">1</span>배 미만 = 순자산보다 싸게 거래 (자산주)
<span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> 높음 = 시장이 자산 이상의 가치를 인정
</code></pre>
<h3>5-2. P/B는 언제 유용한가</h3>
<pre><code class="hljs language-less"><span class="hljs-selector-attr">[P/B 활용이 유효한 섹터]</span>

적합한 섹터:
  ✓ 금융 (은행, 보험): 자산이 수익의 핵심
    <span class="hljs-selector-tag">JPMorgan</span> <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span>: <span class="hljs-number">1.8</span>배 (<span class="hljs-number">2024</span>년)
    <span class="hljs-selector-tag">Bank</span> <span class="hljs-selector-tag">of</span> <span class="hljs-selector-tag">America</span> <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span>: <span class="hljs-number">1.1</span>배
    → 금융주에서 <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">1.0</span> 이하면 저평가 신호

  ✓ 부동산 (REITs): 실물 자산 중심
  ✓ 소재/에너지: 유형 자산 보유

적합하지 않은 섹터:
  ✗ <span class="hljs-selector-tag">IT</span>/소프트웨어: 자산이 주로 무형자산 (브랜드, 특허)
    <span class="hljs-selector-tag">Apple</span> <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span>: <span class="hljs-number">50</span>배+ (자사주 매입으로 자본 작음)
    <span class="hljs-selector-tag">Microsoft</span> <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span>: <span class="hljs-number">13</span>배
    → <span class="hljs-selector-tag">IT</span>기업 <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span>는 높아도 정상, 의미가 낮음
</code></pre>
<h3>5-3. P/B 실전 사례 — 금융 위기 시 활용</h3>
<pre><code class="hljs language-less"><span class="hljs-selector-attr">[2008~2009 금융위기 때 P/B 활용]</span>

<span class="hljs-number">2009</span>년 <span class="hljs-number">3</span>월 (금융위기 저점)
  <span class="hljs-selector-tag">Bank</span> <span class="hljs-selector-tag">of</span> <span class="hljs-selector-tag">America</span>: <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">0.2</span>배
  <span class="hljs-selector-tag">Citigroup</span>: <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">0.3</span>배
  <span class="hljs-selector-tag">JPMorgan</span>: <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">0.6</span>배

의미: 시장이 이 은행들이 파산할 거라 생각
      순자산의 <span class="hljs-number">20</span>~<span class="hljs-number">60%</span>로 거래

결과: 파산하지 않고 회복 →
  <span class="hljs-selector-tag">Bank</span> <span class="hljs-selector-tag">of</span> <span class="hljs-selector-tag">America</span> <span class="hljs-number">5</span>년 후 주가 <span class="hljs-number">10</span>배
  <span class="hljs-selector-tag">JPMorgan</span> <span class="hljs-number">5</span>년 후 주가 <span class="hljs-number">4</span>배

교훈:
  위기 때 <span class="hljs-selector-tag">P</span>/<span class="hljs-selector-tag">B</span> <span class="hljs-number">1</span>배 미만은 강력한 저평가 신호
  단, 파산하지 않는다는 전제가 필수
</code></pre>

<h2>6. DCF (현금흐름 할인법) — 본질적 가치 계산</h2>
<h3>6-1. DCF 개념 이해</h3>
<p>DCF(Discounted Cash Flow)는 <strong>기업이 미래에 창출할 모든 현금흐름을 현재가치로 할인</strong>하여 주식의 내재가치를 계산하는 방법이다. 이론적으로 가장 정확하지만, 가정에 따라 결과가 크게 달라진다.</p>
<pre><code class="hljs language-ini"><span class="hljs-section">[DCF의 핵심 원리]</span>

오늘의 $100 > 1년 후의 $100

왜?
  오늘 $100을 투자하면 1년 후 $110이 된다 (10% 수익률 가정)
  → 1년 후의 $<span class="hljs-attr">110</span> = 오늘의 <span class="hljs-variable">$100</span> (현재가치)
  → 1년 후의 $<span class="hljs-attr">100</span> = 오늘의 <span class="hljs-variable">$90</span>.<span class="hljs-number">9</span> (현재가치)

수식:
  <span class="hljs-attr">PV</span> = FCF₁/(<span class="hljs-number">1</span>+r)¹ + FCF₂/(<span class="hljs-number">1</span>+r)² + ... + FCFₙ/(<span class="hljs-number">1</span>+r)ⁿ

  PV: 현재가치 (적정 주가 × 주식 수)
  FCF: 자유현금흐름 (영업현금흐름 - 설비투자)
  r: 할인율 (가중평균자본비용, WACC)
  n: 예측 기간 (보통 5~10년)
</code></pre>
<h3>6-2. Apple DCF 간이 계산 실습</h3>
<pre><code class="hljs language-bash">[Apple 5년 간이 DCF 계산 (교육 목적)]

기본 데이터 (2024년 기준):
  현재 FCF: <span class="hljs-variable">$1</span>,000억 (연간 약 <span class="hljs-variable">$1</span>,000억 내외)
  예상 FCF 성장률: 연 10% (서비스 성장 반영)
  할인율(r): 9% (WACC 추정)
  터미널 성장률: 3% (장기 성장)
  발행 주식 수: 약 153억 주

5년 FCF 예측:
  Year 1: <span class="hljs-variable">$1</span>,000억 × 1.10¹ = <span class="hljs-variable">$1</span>,100억
  Year 2: <span class="hljs-variable">$1</span>,000억 × 1.10² = <span class="hljs-variable">$1</span>,210억
  Year 3: <span class="hljs-variable">$1</span>,000억 × 1.10³ = <span class="hljs-variable">$1</span>,331억
  Year 4: <span class="hljs-variable">$1</span>,000억 × 1.10⁴ = <span class="hljs-variable">$1</span>,464억
  Year 5: <span class="hljs-variable">$1</span>,000억 × 1.10⁵ = <span class="hljs-variable">$1</span>,611억

현재가치로 할인 (r=9%):
  Year 1: <span class="hljs-variable">$1</span>,100억 / 1.09¹ = <span class="hljs-variable">$1</span>,009억
  Year 2: <span class="hljs-variable">$1</span>,210억 / 1.09² = <span class="hljs-variable">$1</span>,019억
  Year 3: <span class="hljs-variable">$1</span>,331억 / 1.09³ = <span class="hljs-variable">$1</span>,028억
  Year 4: <span class="hljs-variable">$1</span>,464억 / 1.09⁴ = <span class="hljs-variable">$1</span>,037억
  Year 5: <span class="hljs-variable">$1</span>,611억 / 1.09⁵ = <span class="hljs-variable">$1</span>,047억
  5년 FCF 합계: <span class="hljs-variable">$5</span>,140억

터미널 가치 (5년 이후 모든 현금흐름):
  TV = FCF₅ × (1+g) / (r-g)
     = <span class="hljs-variable">$1</span>,611억 × 1.03 / (0.09-0.03)
     = <span class="hljs-variable">$1</span>,659억 / 0.06
     = <span class="hljs-variable">$27</span>,650억

터미널 가치 현재가치:
  TV_PV = <span class="hljs-variable">$27</span>,650억 / 1.09⁵ = <span class="hljs-variable">$17</span>,978억

기업 내재가치 = <span class="hljs-variable">$5</span>,140억 + <span class="hljs-variable">$17</span>,978억 = <span class="hljs-variable">$23</span>,118억
주당 내재가치 = <span class="hljs-variable">$23</span>,118억 / 153억 주 = <span class="hljs-variable">$151</span>

※ 2024년 실제 주가 <span class="hljs-variable">$185</span> → DCF 기준 약 22% 고평가
※ 단, 성장률을 12%로 올리면 <span class="hljs-variable">$190</span>+ 나옴 → 가정이 핵심
</code></pre>
<blockquote>
<p><strong>⚠️ 주의사항</strong>: DCF는 가정에 매우 민감하다. 성장률 1% 변화, 할인율 0.5% 변화가 결과를 20~30% 바꿀 수 있다. DCF는 "정확한 답"이 아니라 "합리적인 범위"를 제시하는 도구로 활용해야 한다.</p>
</blockquote>
<h3>6-3. DCF를 간편하게 역산하는 법</h3>
<pre><code class="hljs language-bash">[역DCF — <span class="hljs-string">"이 주가는 얼마의 성장을 가정하는가"</span>]

역DCF 질문: <span class="hljs-string">"현재 주가가 정당화되려면 몇 % 성장이 필요한가?"</span>

Apple 예시:
  현재 주가: <span class="hljs-variable">$185</span>
  현재 FCF: <span class="hljs-variable">$1</span>,000억
  할인율: 9%, 터미널 성장률: 3%

  역산 결과:
  → <span class="hljs-variable">$185</span>를 정당화하려면 연 FCF 성장률 12% 필요
  → <span class="hljs-string">"Apple이 앞으로 5년간 매년 12% FCF 성장 가능한가?"</span>
  → 이것이 YES/NO의 판단 문제가 된다

  Nvidia 예시:
  현재 주가: <span class="hljs-variable">$800</span> (2024년 고점 기준)
  → 역산 결과: 연 40%+ FCF 성장 필요
  → <span class="hljs-string">"AI 수요가 계속 폭발적으로 성장하면 가능"</span>
  → 이것이 AI 프리미엄 논쟁의 핵심
</code></pre>

<span class="hljs-section">Google: 여러 지표에서 가장 낮은 밸류에이션</span>
 → <span class="hljs-string">"AI 경쟁에서 뒤처질 것"</span> 시장 우려 반영
 → <span class="hljs-string">"싸다"</span> ≠ <span class="hljs-string">"저평가"</span> — 이유를 파악해야 함

<span class="hljs-section">Nvidia: 성장 감안 시 가장 합리적 PEG</span>
 → AI GPU 독점 + 폭발적 이익 성장의 결합

<span class="hljs-section">Apple: ROE 160%+ 압도적 자본 효율성</span>
 → 자사주 매입으로 자기자본 감소 + 높은 이익
 → 성장은 느리지만 주주환원 최강
</code></pre>

<h2>10. 밸류에이션 3대 함정</h2>
<h3>함정 1 — 저PER 함정 (Value Trap)</h3>
<pre><code class="hljs language-markdown">저PER 함정 예시

기업: 미국 대형 백화점 체인 (Macy's, M)
PER: 5배 (엄청나게 싸 보임!)

하지만:
<span class="hljs-bullet">  -</span> 이커머스에 시장 점유율 계속 빼앗김
<span class="hljs-bullet">  -</span> 매출과 이익이 매년 감소 추세
<span class="hljs-bullet">  -</span> 부채 증가, 점포 폐쇄 가속화
<span class="hljs-bullet">  -</span> PER 5배에 샀는데 이익이 반으로 줄면
<span class="hljs-code">    실제로는 PER 10배에 산 셈
</span>
결과: 주가 계속 하락 → "저PER 함정"
</code></pre>
<p><strong>교훈</strong>: PER이 낮은 이유가 "시장이 놓쳤기 때문"인지, "진짜 구조적 문제가 있기 때문"인지 반드시 확인해야 한다.</p>
<h3>함정 2 — 섹터 간 비교 실수</h3>
<pre><code class="hljs language-scss">❌ 잘못된 비교
"JP <span class="hljs-built_in">Morgan</span>(금융) PER <span class="hljs-number">12</span>배 vs Apple PER <span class="hljs-number">29</span>배
 → JP Morgan이 훨씬 싸다!"

이건 사과와 오렌지를 비교하는 것

✅ 올바른 비교
JP Morgan vs Bank of America vs Wells Fargo
(같은 은행 섹터 내 비교)

Apple vs Microsoft vs Google
(같은 빅테크 섹터 내 비교)
</code></pre>
<h3>함정 3 — 일회성 이익 착시</h3>
<pre><code class="hljs language-bash">일회성 이익으로 PER이 낮아 보이는 경우:

기업: XYZ Corp
올해 순이익: <span class="hljs-variable">$5</span>억 (매우 좋아 보임)
주가: <span class="hljs-variable">$50</span>, EPS: <span class="hljs-variable">$5</span>, PER = 10배 (저렴해 보임!)

그런데:
  정상 영업이익: <span class="hljs-variable">$2</span>억
  공장 매각 이익 (일회성): <span class="hljs-variable">$3</span>억
  ──────────────────────────
  총 순이익: <span class="hljs-variable">$5</span>억

조정 EPS: <span class="hljs-variable">$2</span> (일회성 제거)
조정 PER: <span class="hljs-variable">$50</span> ÷ <span class="hljs-variable">$2</span> = 25배 (전혀 싸지 않음)

확인 방법:
  → Non-GAAP EPS (조정 EPS) vs GAAP EPS 비교
  → 일회성 항목을 제거한 <span class="hljs-string">"정상 이익"</span> 기준 분석
</code></pre>

<h2>11. 실전 밸류에이션 체크리스트 (8개 항목)</h2>
<p>투자 결정 전 반드시 이 8개 항목을 체크하자.</p>
<pre><code class="hljs language-sql">[밸류에이션 최종 체크리스트]

체크 <span class="hljs-number">1</span>: 섹터 대비 <span class="hljs-keyword">PER</span> 위치는?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ 해당 섹터 평균 <span class="hljs-keyword">PER</span> 확인

□ 그 차이를 정당화할 근거가 있는가?

체크 <span class="hljs-number">2</span>: PEG로 성장 감안 시 합리적인가?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ PEG <span class="hljs-number">1.5</span> 이하: 성장 대비 합리적
□ PEG <span class="hljs-number">2.5</span> 이상: 고평가 주의
□ EPS 성장률이 지속 가능한가?

체크 <span class="hljs-number">3</span>: <span class="hljs-keyword">Trailing</span> <span class="hljs-keyword">PER</span> vs Forward <span class="hljs-keyword">PER</span> 차이는?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Forward <span class="hljs-keyword">PER</span>이 크게 낮으면 이익 급증 기대
□ 그 기대가 현실적인가?
□ 애널리스트 컨센서스 EPS 성장률 확인

체크 <span class="hljs-number">4</span>: 부채가 많다면 EV<span class="hljs-operator">/</span>EBITDA 확인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Debt<span class="hljs-operator">/</span>EBITDA <span class="hljs-number">5</span>배 이상이면 위험
□ EV<span class="hljs-operator">/</span>EBITDA가 섹터 평균 내에 있는가?
□ 현금흐름으로 이자 충분히 감당 가능한가?

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

□ FCF가 순이익의 <span class="hljs-number">70</span><span class="hljs-operator">%</span> 이상인가?

□ 현재 <span class="hljs-keyword">PER</span>이 역사적 범위의 어디에 있나?
□ 과거 고<span class="hljs-keyword">PER</span> 시기와 현재 차이점은?

<h2>12. 밸류에이션 실전 5단계 워크플로우</h2>
<pre><code class="hljs language-makefile">[밸류에이션 분석 5단계]

<span class="hljs-section">1단계: 섹터 확인</span>
  → 해당 섹터의 평균 PER/EV/EBITDA 파악
  → 경기 사이클에서 이 섹터의 위치

<span class="hljs-section">2단계: PER 체크</span>
  → Trailing PER이 섹터 평균보다 높은가 낮은가?
  → Forward PER과의 차이 (성장 기대치 확인)

<span class="hljs-section">3단계: 성장성 확인</span>
  → PEG &#x3C; 1.5 이하인가?
  → EPS 성장률의 질(매출 성장 vs 비용 절감) 확인

<span class="hljs-section">4단계: 재무 건전성</span>
  → 부채 많으면 EV/EBITDA로 재확인
  → FCF가 순이익과 함께 증가하는가?

<span class="hljs-section">5단계: 경쟁사 비교</span>
  → 동일 섹터 내 최소 3개 기업과 비교
  → 고평가라면 그 이유(해자·성장)가 명확한가?
</code></pre>

<h2>마무리 — 밸류에이션은 과학이 아닌 예술</h2>
<p>밸류에이션 지표들은 "이 주가가 싼지 비싼지"를 판단하는 <strong>도구</strong>다. 절대적 답이 아니다.</p>
<ul>
<li>PER 5배도 함정일 수 있고 (가치 함정)</li>
<li>PER 100배도 정당화될 수 있다 (고성장 + 해자)</li>
</ul>
<p>중요한 것은 <strong>왜 이 밸류에이션인가</strong>를 이해하는 것이다. 그 "왜"를 찾는 여정이 바로 투자 분석의 본질이다.</p>
<pre><code class="hljs language-scss"><span class="hljs-selector-attr">[밸류에이션 핵심 공식]</span>

합리적 투자 =
  현재 밸류에이션 지표 분석
  + 미래 성장성 예측 (PEG, DCF)
  + 경제적 해자 강도 평가
  + 섹터/거시 환경 고려

절대 하지 말아야 할 것:
  ✗ <span class="hljs-string">"이미 많이 올랐으니 비쌀 것"</span> (전제 없는 판단)
  ✗ <span class="hljs-string">"PER이 낮으니 싸다"</span> (섹터 무시)
  ✗ <span class="hljs-string">"유명한 회사니까 괜찮겠지"</span> (분석 생략)
</code></pre>
<p>다음 편에서는 이 밸류에이션이 정당화되는 조건, 즉 **성장성과 경제적 해자(Economic Moat)**를 깊이 파헤쳐보자.</p>
