<p>투자에서 가장 어려운 두 가지를 꼽으라면 "언제 사는가"와 "언제 파는가"입니다. 특히 매도는 더 어렵습니다. 이번 편에서는 기술적 분석의 핵심 도구들과 공포&#x26;탐욕지수, 분할 매수 전략, 손절 원칙, 피라미딩까지 체계적으로 정리합니다.</p>
<hr>
<h2>1. 기술적 분석 vs 펀더멘털 분석</h2>
<p>두 분석 방법은 서로 경쟁하는 것이 아니라, <strong>상호 보완적</strong>입니다.</p>

<h2>4. RSI (상대강도지수)</h2>
<p>RSI(Relative Strength Index)는 최근 주가 상승과 하락의 강도를 0~100 사이 숫자로 나타냅니다.</p>
<h3>RSI 구간 시각화</h3>
<pre><code class="hljs language-scss">RSI <span class="hljs-number">0</span>~<span class="hljs-number">100</span> 구간

<span class="hljs-number">100</span> │──────────────── 극단적 과매수 (버블 신호)
 <span class="hljs-number">80</span> │
 <span class="hljs-number">70</span> │──────────────── 과매수 (Overbought) → 매도 고려
    │
 <span class="hljs-number">50</span> │──────────────── 중립선 (상승/하락 추세 구분)
    │
 <span class="hljs-number">30</span> │──────────────── 과매도 (Oversold) → 매수 고려
 <span class="hljs-number">20</span> │
  <span class="hljs-number">0</span> │──────────────── 극단적 과매도 (공황 신호)

  해석:
  RSI > <span class="hljs-number">70</span>: <span class="hljs-string">"너무 많이 올랐다"</span> → 단기 조정 가능성
  RSI <span class="hljs-number">40</span>~<span class="hljs-number">60</span>: 중립 구간
  RSI &#x3C; <span class="hljs-number">30</span>: <span class="hljs-string">"너무 많이 떨어졌다"</span> → 반등 가능성

<h3>방법 1: 기계적 손절 (−8% 룰)</h3>
<p>윌리엄 오닐(William O'Neil, CAN SLIM 창시자)이 제안한 방법입니다.</p>
<pre><code class="hljs language-bash">-8% 기계적 손절 규칙

- 감정 배제 → 더 큰 손실 방지
- -8% 손절 후 다른 종목에서 회복 가능
- -50% 버티기보다 -8% × 여러 번이 낫다

수학:
  -8% 손절 후 회복 필요 수익률: +8.7%  (쉬움!)
  -50% 버티기 후 회복 필요 수익률: +100% (극도로 어려움)
</code></pre>

<h2>9. 거래량: 신뢰도를 높이는 지표</h2>
<p>차트에서 가격만 보면 안 됩니다. <strong>거래량은 가격 움직임의 신뢰도를 보여줍니다.</strong></p>
<pre><code class="hljs language-markdown">거래량과 주가의 4가지 조합

조합 1: 주가↑ + 거래량↑ = 강한 상승 신호 (매수세 강함)
<span class="hljs-code">        거래량: ████████████████ (평균보다 높음)
        주가: ↑↑↑↑↑
        의미: 기관 투자자가 적극 매수 중
</span>
조합 2: 주가↑ + 거래량↓ = 약한 상승 (속임수 가능성)
<span class="hljs-code">        거래량: ████ (평균보다 낮음)
        주가: ↑↑↑
        의미: 소수 세력의 인위적 상승 가능, 주의!
</span>
조합 3: 주가↓ + 거래량↑ = 강한 하락 신호 (매도 고려)
<span class="hljs-code">        거래량: ████████████████ (평균보다 높음)
        주가: ↓↓↓↓↓
        의미: 기관이 대규모 매도 중, 추세 하락 신호
</span>
조합 4: 주가↓ + 거래량↓ = 약한 하락 (조정 후 반등 가능성)
<span class="hljs-code">        거래량: ████ (평균보다 낮음)
        주가: ↓↓↓
        의미: 일시적 조정일 수 있음, 매도 서두르지 말 것
</span></code></pre>

<h2>10. 피라미딩 (Pyramiding): 수익 나는 포지션 늘리기</h2>
<p>피라미딩은 <strong>이미 수익이 나고 있는 포지션을 단계적으로 늘리는</strong> 투자 기법입니다. 손실 나는 포지션에 추가 매수하는 "물타기"와 반대 개념입니다.</p>
<h3>피라미딩 원리</h3>
<pre><code class="hljs language-bash">피라미딩 vs 물타기 비교

물타기 (Wrong Way):                피라미딩 (Right Way):
매수가 <span class="hljs-variable">$100</span>                        매수가 <span class="hljs-variable">$100</span>
주가 <span class="hljs-variable">$80</span>으로 하락 (-20%)           주가 <span class="hljs-variable">$120</span>으로 상승 (+20%)
→ 추가 <span class="hljs-variable">$100</span> 매수 (평단낮추기)      → 확인 후 추가 <span class="hljs-variable">$50</span> 매수
주가 <span class="hljs-variable">$60</span>으로 추가 하락             주가 <span class="hljs-variable">$140</span>으로 추가 상승
→ 또 추가 매수...                 → 또 추가 <span class="hljs-variable">$25</span> 매수
결과: 손실 눈덩이                  결과: 수익 극대화

원칙: 시장이 <span class="hljs-string">"맞다"</span>고 증명했을 때만 베팅 키우기
</code></pre>
<h3>피라미딩 실전 전략</h3>
<pre><code class="hljs language-bash">피라미딩 3단계 예시 (총 <span class="hljs-variable">$10</span>,000 투자)

1단계: 초기 매수 (리스크 작게)
  주가: <span class="hljs-variable">$100</span>
  투자: <span class="hljs-variable">$4</span>,000 → 40주 매수
  근거: 기술적 패턴 형성 + 펀더멘털 좋음

2단계: 상승 확인 후 추가 매수
  주가: <span class="hljs-variable">$120</span> (+20%)
  투자: <span class="hljs-variable">$3</span>,000 → 25주 추가 매수 (2단계)
  근거: 200일 MA 위 + 실적 서프라이즈 확인
  누적: 65주, 평균 단가 <span class="hljs-variable">$107</span>.7

3단계: 추세 강화 확인 후 마지막 추가
  주가: <span class="hljs-variable">$140</span> (+40%)
  투자: <span class="hljs-variable">$2</span>,000 → 14.3주 추가 매수
  근거: 골든크로스 + 기관 매수 증가
  누적: 79.3주, 평균 단가 <span class="hljs-variable">$126</span> (현재가 <span class="hljs-variable">$140</span>, 이미 +11%)

나머지 <span class="hljs-variable">$1</span>,000: 비상금/손절 여유
────────────────────────────────────
총 투자: <span class="hljs-variable">$9</span>,000 (1,000은 여유)
보유 주식: 79.3주
평균 단가: ~<span class="hljs-variable">$113</span>.5

만약 <span class="hljs-variable">$160</span>까지 상승하면:
  평가액: 79.3 × <span class="hljs-variable">$160</span> = <span class="hljs-variable">$12</span>,688
  수익률: +41% (원금 <span class="hljs-variable">$9</span>,000 대비)
</code></pre>

<h2>13. 실전 시나리오 3가지</h2>
<p>가장 자주 접하는 투자 상황을 구체적으로 다룹니다.</p>
<h3>시나리오 1: "NVDA를 $400에 샀는데 $300으로 떨어졌다. 어떻게?"</h3>
<pre><code class="hljs language-bash">상황:
  매수가: <span class="hljs-variable">$400</span>
  현재가: <span class="hljs-variable">$300</span> (-25% 손실)

질문 1: NVDA의 투자 논리(thesis)는 여전히 유효한가?
  → AI GPU 수요, 데이터센터 성장, CUDA 생태계 독점
  → 만약 이 논리가 유지되고 있다면?

질문 2: 왜 <span class="hljs-variable">$300</span>으로 떨어졌는가?
  A. 시장 전체 하락 (S&#x26;P 500도 -20%)
     → NVDA만의 문제 아님 → 보유 or 추가 매수 검토
  B. NVDA 자체 악재 (실적 쇼크, 경쟁자 등장)
     → 투자 논리 훼손 → 손절 검토

결정:
  [A의 경우]
  ① 손절 기준(-25%)에 도달했으므로 기계적 손절 원칙이라면 매도
  ② 또는 분할 추가 매수 (비중 5% 이하 유지 조건)
     <span class="hljs-variable">$300</span>에 추가 매수 → 평균 단가 <span class="hljs-variable">$350</span>으로 낮춤

  [B의 경우]
  → 추가 하락 전에 즉시 손절! 더 큰 손실 방지

핵심 교훈: 감정이 아닌 <span class="hljs-string">"논리 유효성"</span>으로 판단하라
</code></pre>
<h3>시나리오 2: "AAPL이 +30% 올랐다. 지금 팔아야 하나?"</h3>
<pre><code class="hljs language-ini">상황:
  매수가: $150
  현재가: $195 (+30% 수익)

판단 기준 체크리스트:

펀더멘털:
  <span class="hljs-section">[✓]</span> 실적 계속 성장 중? → 보유 이유 있음
  <span class="hljs-section">[?]</span> 현재 PER이 역사적 고점? → PER 30배 = 고평가 경계선
  <span class="hljs-section">[✓]</span> AI 서비스 사업 아직 성장 초기? → 성장 스토리 살아있음

기술적:
  <span class="hljs-section">[?]</span> RSI가 75 이상? → 단기 과매수, 조정 가능성
  <span class="hljs-section">[✓]</span> 200일 MA 위 유지? → 상승 추세 유효
  <span class="hljs-section">[?]</span> F&#x26;G 지수 80+? → 시장 전체 과열

결정:
  → 1/3 매도 (수익 확정): $195에 33주 매도
  → 나머지 2/3 보유: 트레일링 스탑 -15% 설정 ($195 × <span class="hljs-attr">0.85</span> = <span class="hljs-variable">$165</span>)
  → 결과: 수익의 일부 확정 + 추가 상승 참여

핵심 교훈: 완벽한 고점은 알 수 없다. 분할 매도로 후회 최소화
</code></pre>
<h3>시나리오 3: "시장 전체가 -20% 떨어졌다. 어떻게?"</h3>
<pre><code class="hljs language-yaml"><span class="hljs-string">상황:</span>
  <span class="hljs-string">S&#x26;P</span> <span class="hljs-number">500</span> <span class="hljs-string">기준</span> <span class="hljs-number">-20</span><span class="hljs-string">%</span> <span class="hljs-string">하락</span> <span class="hljs-string">(베어마켓</span> <span class="hljs-string">진입</span> <span class="hljs-string">기준)</span>
  <span class="hljs-string">보유</span> <span class="hljs-string">포트폴리오도</span> <span class="hljs-number">-15</span><span class="hljs-string">~20%</span> <span class="hljs-string">하락</span>
  <span class="hljs-string">뉴스:</span> <span class="hljs-string">경기침체</span> <span class="hljs-string">우려,</span> <span class="hljs-string">공포</span> <span class="hljs-string">극심</span>

<span class="hljs-string">감정</span> <span class="hljs-string">vs</span> <span class="hljs-string">이성:</span>
  <span class="hljs-string">감정:</span> <span class="hljs-string">"더 떨어지기 전에 팔아야 해!"</span>
  <span class="hljs-string">이성:</span> <span class="hljs-string">역사적으로</span> <span class="hljs-number">-20</span><span class="hljs-string">%는</span> <span class="hljs-string">'매수 기회'</span>
